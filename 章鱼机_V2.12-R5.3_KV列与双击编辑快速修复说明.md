# 章鱼机 V2.12-R5.3 KV 列与双击编辑快速修复说明

## 修复范围

本修包基于 V2.12-R5.2，只处理以下问题：

1. KV 记忆表字段名称列不再使用百分比或固定大宽度，而是根据当前表可见字段中最长名称计算。
2. 所有记忆表取消“单击选中、Enter 编辑、长按编辑”的主交互，统一为电脑双击、手机双点进入或结束编辑。
3. Git 快速恢复不再把 404、旧格式或损坏文件直接当成可恢复备份；恢复前逐个校验候选，自动跳过无效项。

## KV 字段名称列

新增共享模块 `field_width.js`，结构编辑器与 KV 表共用同一套字符视觉宽度算法。

KV 表宽度范围：

- 桌面：64–116px
- 手机：48–76px

宽度依据当前表实际可见字段名计算。中文、英文、数字和下划线采用不同视觉权重。超长名称允许自然换行，不再长期占据手机三分之一宽度。

生产浏览器模块核验样本“双方_核心关系定义”计算结果：

- 桌面：88px
- 手机：52px
- 390px 视口下正文列约 288px
- 页面级横向溢出：0

## 编辑交互

所有 KV 表和行式表使用相同规则：

- 单击：不改变记忆表会话状态，不进入编辑。
- 电脑双击：进入或结束当前字段/行编辑。
- 手机双点：在 360ms 内对同一字段或行双点，进入或结束编辑。
- Esc：退出编辑并清除焦点状态。
- F2：保留为键盘无障碍辅助入口，不在界面中作为主操作提示。

页面提示已改为“雙击编辑 / 手机双点 / Esc 退出”，原“单击选中 / 长按编辑 / Enter 编辑”提示和逻辑已移除。

## Git 快速恢复

修复内容：

- GitHub 文件按路径段编码，避免目录斜杠被整体编码。
- 使用 raw 内容下载，兼容 Contents API 不返回大文件 `content` 的情况。
- 单文件和分片备份统一先调用 `BackupService.parseAndValidate()`。
- 最新候选遇到 404、文件缺失、旧备份格式或损坏时，自动尝试下一个较旧候选。
- 所有候选都不可用时，返回明确的跳过数量和原因摘要，不再只显示“文件不是有效的当前版完整备份”。

## 数据兼容

本修包未修改：

- 记忆 schema
- `.ee` 备份格式
- 标签、关系、生命周期与更新历史
- 表格虚拟化、增量保存和撤销机制
