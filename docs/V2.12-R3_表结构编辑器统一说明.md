# 章鱼机 V2.12-R3 表结构编辑器统一说明

## 版本目标

本版只处理 V2.12 规划中的第三阶段：将分散的“模板可视化编辑、模板 JSON 编辑、字段编辑、表级设置”统一为一个表结构编辑器。

不改动记忆记录数据，不改变 `.ee` 备份格式，不提前实施最终领域拆分与架构门禁。

## 用户侧变化

### 单一入口

管理页与当前表工具栏统一使用“表结构”入口。原有“可视化编辑”和“JSON 编辑”不再分别打开两个弹窗。

模板库中的每个模板也只保留一个“编辑结构”按钮。

### 三个互通视图

表结构编辑器包含三个标签页：

1. **字段**：编辑字段分组、字段名、类型、默认值、普通模式显示、AI 编辑权限、摘要标签和高级字段设置。
2. **表格**：编辑八张表的名称、模式、记忆层级、自动更新、触发方式、注入方式和表级高级策略。
3. **结构 JSON**：默认按“路径 / 类型 / 值”的结构表编辑同一份 schema；原始 JSON 仅保留在高级折叠区用于导入和核对。

三个视图使用同一份内存草稿，不存在字段视图和 JSON 视图各自保存一份配置的问题。

### 字段分组

字段页按分组生成一级分区行。例如真实“核心确认档案”会显示：

- 角色设定迁移区
- 核心确认

修改字段的分组名称后，下一次渲染会自动归入对应分组。

### JSON 不再是独立世界

结构 JSON 页将模板、表格、更新策略、注入策略和字段属性展开为可编辑路径表。用户不必直接处理一整块 JSON 文本。

原始 JSON 仍可展开：

- “用当前结构刷新”用于查看当前草稿；
- “应用原始 JSON”用于高级导入；
- 导入后会重新规范化，并回到同一个 schema 草稿。

## 代码结构

### 新增 `schema_model.js`

只处理纯结构规则：

- 模板草稿准备与规范化；
- 模板、表、字段统计；
- 字段分组；
- schema 路径读取和写入；
- 路径值类型解析；
- 表格/字段增删和移动；
- 原始 JSON 导入。

该模块不操作 DOM。

### 新增 `schema_editor.js`

只处理表结构编辑器的视图模型与渲染：

- 字段页；
- 表格页；
- 结构 JSON 路径表；
- 编辑控件值同步。

具体模板保存仍复用原有模板持久化和数据迁移流程。

### 新增 `memory_schema_editor.css`

编辑器使用单层大工作区：

- 顶部模板摘要；
- 一个标签栏；
- 一个当前视图区域；
- 底部保存区。

宽表只在自身容器内滚动，手机端使用全屏工作区，不继续压缩主记忆表。

### 主控制器收敛

原来的两套模板弹窗渲染和监听已从 `memory_table.js` 移除。主控制器只负责：

- 打开和关闭统一编辑器；
- 保存规范化后的模板；
- 协调当前标签页和当前表索引。

`memory_table.js` 当前为 3,872 行，低于本版 3,900 行门禁。

## 数据兼容性

本版保持：

- `memoryTableTemplates` 数据结构不变；
- 模板 ID、表 ID 和字段 ID 继续沿用；
- 现有 updatePolicy 和 injectionPolicy 继续使用；
- 模板绑定关系不变；
- 模板导入、导出和记忆包导出仍可使用；
- `.ee` 备份与恢复格式不变；
- 角色现有八表数据不因打开编辑器而复制或迁移。

保存时仍调用原有模板规范化和模板数据同步流程。

## 真实数据核验

数据来源：`章鱼喷墨_备份数据_2026-07-21_清理优化版.ee` 中的真实 `memoryTableTemplates`。

模板：分层可检索角色记忆 V2（含原数据）。

| 指标 | 结果 |
|---|---:|
| 表格 | 8 张 |
| 字段 | 94 个 |
| 字段组 | 31 个 |
| 字段页标签 | 3 个统一视图标签 |
| 旧独立模板弹窗 | 0 |
| 结构 JSON 路径项 | 1,352 项 |
| 结构 JSON 分区 | 103 个 |

互通核验：

1. 在字段页将第一个字段名改为 `user_人格底色_结构核验`；
2. 切换结构 JSON 后，对应路径 `tables.0.columns.0.key` 显示相同值；
3. 在高级原始 JSON 中修改模板名称并应用；
4. 返回字段页，模板名称同步显示为修改后的值。

页面尺寸：

- 桌面：1440 / 1440，无页面级横向溢出；
- 手机：390 / 390，无页面级横向溢出；
- 表格宽度超出时，只在编辑器表格内部滚动。

## 已知边界

- 结构 JSON 路径项较多，按分区折叠显示；本版没有引入路径搜索，后续可在不改变 schema 的前提下增加。
- 字段高级设置仍使用折叠区，避免普通用户首屏看到过多低频配置。
- 本版统一了编辑入口和数据模型，但还没有完成最终的领域目录迁移；该工作属于下一阶段代码领域拆分与门禁。
