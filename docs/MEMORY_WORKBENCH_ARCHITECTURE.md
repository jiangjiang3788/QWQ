# 记忆 UI 工作台（V2.9-R2）

## 用户入口

结构记忆 App 只保留三个一级入口：

- 记忆：当前状态、活跃待办和结构化记忆内容。
- 待处理：更新审核、短期候选、可靠性复核、使用反馈和失败任务。
- 管理：模板、召回诊断、质量测试、历史、导入导出和高级整理设置。

旧的十个技术标签仍保留在隐藏兼容容器中，不参与普通 UI 展示。

## UI 边界

`workspace.js` 负责：

- 一级入口与内部视图映射；
- 待处理数量；
- 当前状态与活跃待办摘要；
- 待处理和管理概览卡片；
- 详情页标题与返回路径。

`memory_table.js` 继续负责：

- 数据读取和页面总调度；
- 旧视图渲染器的兼容调用；
- 原事件处理和业务服务连接。

后续版本可以继续把具体列表视图从 `memory_table.js` 拆到 `ui/`，不再改变一级信息架构。

## 层级约束

- 一级：记忆 / 待处理 / 管理。
- 二级：概览卡片进入单个功能页。
- 临时层：编辑、确认和导入弹窗。

不增加三级标签或嵌套折叠导航。高级设置只保留一层 `<details>`。
