# 章鱼机 V2.14-R0：记忆完整性医生与导入安全说明

## 一、版本定位

V2.14-R0 是 2.14 系列的第一版，基于 V2.13-R5.4。它不继续扩张自动写入能力，而是先建立记忆数据的安全底线：

1. 系统能够只读检查自己的结构和引用是否闭合；
2. 可迁移记忆包导入新角色时，内部 ID、跨表关系和晋升目标能够一致重映射；
3. 源角色的消息游标、轮次、审核任务和运行历史不会被带入目标角色。

本版不包含自动修复、统一事务总线、字段级写入策略和检索纯函数化；这些仍按后续计划逐版实施。

## 二、记忆完整性医生

入口位于：

```text
记忆 → 管理 → 记忆完整性医生
```

扫描是只读操作，不会保存角色，也不会修改模板、正式记忆、候选、审核或运行状态。

### 检查范围

- 模板、表格、字段和记忆行 ID 重复；
- 角色绑定模板失效及孤立模板数据；
- 当前角色绑定范围内的唯一表格职责重复；
- 长期候选未配置目标、目标不存在、指向自身或目标类型错误；
- 字段锁定、KV 值和行单元格指向已删除字段；
- 内容完全重复的行记忆；
- 记忆关系指向不存在的目标；
- 长期晋升追踪断开；
- 正式记忆保留其他聊天或已删除消息的来源引用；
- 旧版 `processed` 候选和已保存候选的目标断开；
- 候选引用不存在的聊天轮次；
- 孤立表格运行状态、失效消息游标和轮次游标；
- 表格等待不存在的审核批次；
- 审核草案目标表不存在；
- 生命周期维护从未运行。

### 报告结果

报告提供：

- 0–100 完整性分数；
- 严重、高风险、需处理、提示和信息五个等级；
- 每个问题的原因、作用范围和建议；
- JSON 报告导出。

R0 只负责发现问题，不提供“一键自动修复”。这是为了避免在迁移器和统一事务尚未完成前，扫描模块成为新的隐式写入入口。

## 三、可迁移记忆包导入

V2.14-R0 将当前导出的记忆包标记为：

```json
{
  "schemaVersion": "2.8",
  "producerVersion": "2.14-R0",
  "packageProfile": "portable_snapshot"
}
```

`schemaVersion` 暂时维持 2.8，避免在统一迁移注册表完成前伪造一次不完整的大版本迁移；`producerVersion` 用于标记当前生成器能力。

### 导入前预览

导入记忆包时会显示：

- 模板、表格、字段和行记忆数量；
- 短期候选数量；
- 将重新生成和重映射的引用；
- 将清空的源角色运行状态。

选择导入记忆时执行完整安全迁移；只导入模板结构时不写入当前角色记忆。

### 全局 ID 映射

导入计划一次性建立：

```text
模板 ID 映射
表格 ID 映射
字段 ID 映射
行 ID 映射
```

行映射覆盖整个导入包，而不是只在当前表内查找。因此以下跨表引用可以同步迁移：

- 长期候选的 `promotionPolicy.targetTableId`；
- 行关系 `supersedes / supersededBy / conflictsWith / relatedTo`；
- 审核行中的正式长期目标；
- Sidecar 候选的目标模板、表格和行；
- 质量测试的预期表格和预期行；
- `事件ID / 记录ID / 记忆ID / 原始记录ID / 来源记录ID` 等内部稳定记录标识。

只有精确匹配到导入包内唯一旧行 ID 的结构化 ID 字段才会重映射，普通文本不会被全文替换。

### 跨角色运行状态重置

以下内容不会继承：

- 表格消息游标和轮次游标；
- 等待审核批次；
- 当前活动轮次及轮次历史；
- Sidecar 来源轮次、来源消息、操作 ID 和历史；
- 正式行 `sourceMessageIds`；
- 检索向量和索引时间；
- 审核队列和后台任务历史；
- 召回反馈事件与质量运行历史；
- 生命周期上次维护结果。

正式记忆内容、字段锁定、可验证的关系和稳定设置会保留。

### 无法验证的旧候选

已标记为 `processed / promoted / merged`，但导入后无法找到正式目标的候选，不会继续冒充已保存，而会转换为：

```text
legacy_unverified
```

用户之后可以重新保存、忽略或删除。

## 四、实际记忆包验证

使用 `阿沉_memory_package_逻辑收敛修正版.json` 进行只读验证：

- 1 个模板；
- 8 张表；
- 94 个字段；
- 234 条行记忆；
- 导入后仍为 234 条；
- 正式行来源消息引用剩余 0；
- 结构化旧 ID 精确引用剩余 0；
- 4 条 Sidecar 候选被迁移；
- Sidecar 历史、质量运行历史均被清空；
- 源文件对象未发生修改。

只读医生在该旧包中发现 15 项问题，主要为旧轮次游标、旧候选、候选轮次引用，以及长期候选未配置显式晋升目标。这些问题只进入报告，本版不会擅自修改源包。

详细数据见：

```text
docs/V2.14-R0_实际记忆包完整性与导入安全验证.json
```

## 五、架构边界

新增两个叶子模块：

```text
js/features/memory/integrity_doctor.js
js/features/memory/package_adapter.js
```

归属：

- `integrityDoctor` → 治理领域；
- `packageAdapter` → 基础领域。

主控制器只负责入口编排和用户确认，不复制扫描或映射算法。正式记忆写入体系没有在本版另建第二套数据层。

## 六、后续版本

下一阶段按既定顺序推进：

1. 统一角色级事务与持久化协调；
2. 正式写入门禁；
3. 字段级证据与写入策略；
4. 用户正式状态与角色运行态分离；
5. 检索纯函数化；
6. 统一 Schema 迁移注册表。
