# V2.14 记忆系统完整计划与进度

更新时间：V2.14-R8.1

## 总体目标

将结构化记忆系统收敛为一个可信、可解释、可回滚、单人使用不过度设计的私人档案系统。

原则：

1. 正式记忆、候选、审核和运行态分开；
2. 正式写入只有一个受保护入口；
3. 读取记忆不能偷偷改正式档案；
4. 自动更新必须能解释最终策略和来源；
5. 导入、审核和晋升失败时不能留下半完成状态；
6. 优先解决高风险断点，再做体验和维护功能；
7. 单人自用，不增加多人锁、远程冲突和企业级权限系统。

## V2.14-R8.1 修正版说明

R8.1 不新增计划项，因此总体完成项仍为 13 项。它修复真实备份审计发现的策略闭环断点：

1. Sidecar 当前状态改为逐字段证据和置信度；
2. 当前状态推断只进入会话运行态；
3. 旧当前状态重复审核按字段保留最新值并清理旧批次；
4. 当前角色的表级直接写入覆盖真正作用到明确候选通道；
5. 日常观察按日期 Upsert；
6. 设置页显示字段实际路由分布；
7. runtime-only 旧正式值不再进入上下文。

字段结构审计已经形成迁移建议，但 R8.1 不自动删除、合并或重排用户现有字段。

## 下一版 V2.14-R9

范围：

1. 将完整性扫描、过期、冲突、归档和合并建议收敛为“预览—确认—执行—回执”闭环；
2. 生命周期维护只在用户主动运行或低频维护时执行，不在正常聊天召回中修改正式记忆；
3. 每条记忆统一展示来源消息、来源候选、写入原因、首次/最近出现时间；
4. 展示历次字段变化、合并来源、替代关系和最近召回；
5. 复用 WorkItem、WriteGateway、recordKey 和统一回执，不再建立平行治理系统；
6. 不在 R9 同时启动浏览器 E2E 和旧兼容层删除。
