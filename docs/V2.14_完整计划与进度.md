# V2.14 记忆系统完整计划与进度

更新时间：V2.14-R9

## 总体目标

将结构化记忆系统收敛为可信、可解释、可回滚、不过度设计的单人私人档案系统。

原则：

1. 正式记忆、候选、审核和运行态分开；
2. 正式写入只有一个受保护入口；
3. 读取记忆不能偷偷修改正式档案；
4. 自动更新必须能解释最终策略、证据和来源；
5. 导入、审核、晋升和维护失败时不能留下半完成状态；
6. 优先解决真实断点，再增加产品功能；
7. 单人自用，不增加多人锁、远程冲突和企业级权限系统。

## V2.14-R8.1 修正版

R8.1 不新增计划完成项，修复真实备份审计发现的直接写入分流断点：

1. Sidecar 逐字段证据和置信度；
2. 当前状态推断进入运行态；
3. 重复审核按字段保留最新值；
4. 表级直接覆盖作用到明确候选；
5. 日常观察按日期 Upsert；
6. 设置页显示字段实际路由；
7. runtime-only 旧值停止注入上下文。

## V2.14-R9 完成内容

1. 生命周期健康中心和只读维护计划；

4. 不自动删除或自动合并正文；
5. 可能重复组进入人工行检查器；
6. 来源和变化事件统一协议；
7. 正式创建、字段更新、Upsert、确认、关系和合并写入变化链；
8. 旧来源和旧 versionLog 可只读派生；
9. 迁移快照重映射变化链中的行引用并清除源聊天引用；
10. Schema 3.0→3.1；
11. Proment 恢复到桌面应用网格。
