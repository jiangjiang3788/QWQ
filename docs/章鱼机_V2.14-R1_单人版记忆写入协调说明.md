# 章鱼机 V2.14-R1：单人版记忆写入协调说明

## 一、版本定位

V2.14-R1 基于 V2.14-R0，集中解决“同一个角色的多个记忆入口各自保存、失败恢复范围不一致”的问题。

本项目按单人自用设计。本版不引入多人锁、远程版本号、分布式事务或复杂冲突合并，只提供足够覆盖本地单用户场景的能力：

- 同一角色的正式记忆写入按顺序执行；
- 写入前保存完整 `memoryTables` 快照；
- 持久化失败时恢复正式档案、候选、审核、游标、任务和历史；
- 无实际变化时不写磁盘；
- 保留幂等、操作回执和撤销能力。

## 二、统一写入协调器

新增：

```text
js/features/memory/write_coordinator.js
```

标准流程：

```text
等待同角色前一项写入完成
→ 捕获 memoryTables 快照
→ 执行业务变更
→ 无变化则直接结束
→ 持久化角色
→ 成功返回 transactionId
→ 失败恢复快照并尝试持久化恢复结果
```

协调器不取代现有 `Domain`。`Domain` 仍是正式记忆的数据操作层；协调器只负责用例级串行、保存和回滚。

## 三、已接入的高风险写入链

本版统一接入：

- 表格单字段编辑；
- 标签编辑；
- 全屏整行编辑；
- 表格撤销；
- 行新增、删除和移动；
- 聊天 Sidecar 自动更新；
- Sidecar 候选保存、合并、忽略和删除；
- AI 直接更新正式表；
- 更新审核接受、拒绝、取消和回滚；
- 长期候选批准晋升；
- 日记文本转换为档案表格。

这些入口现在使用同一类完整快照和持久化失败恢复规则。

## 四、撤销链补全

整行编辑原先能够保存，但撤销解析仍按“必须有单字段 ID”的旧规则判断，导致整行撤销找不到目标。

V2.14-R1 将撤销目标区分为：

- 单字段编辑：必须解析字段；
- 标签编辑：只需解析表和行；
- 整行编辑：解析表和行，再按保存的字段集合恢复。

整行编辑、保存失败回滚和成功后撤销均已在实际记忆包上验证。

## 五、单人版刻意不做的能力

本版不增加：

- 多设备同时编辑锁；
- 服务端 revision/CAS；
- 多人协作权限；
- 三方合并和冲突解决器；
- 跨浏览器远程事务恢复。

对于当前单人、本地使用场景，这些能力成本高且收益有限。后续只有在真实出现跨设备覆盖问题时再增加轻量 revision。

## 七、后续边界

本版优先统一高风险正式写入链。模板设置、低风险运行统计、生命周期扫描和部分兼容元数据仍保留原保存方式，后续应结合字段级写入策略和检索纯函数化逐步收敛，避免一次性改造过大。
