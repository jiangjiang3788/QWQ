# 章鱼机 V2.14-R6：主控制器用例拆分与模块内聚说明

## 目标

V2.14-R5 后，`js/modules/memory_table.js` 已达到 3989 行，接近 4000 行架构门禁。继续在主控制器内增加功能，会让召回、审核、导入导出和页面控制互相耦合。

R6 不新增用户功能，重点是把已经稳定的用例从主控制器中移出，同时保持数据格式、按钮行为、操作回执和失败回滚不变。

## 本版拆分

### 1. 召回编排

新模块：

```text
js/features/memory/retrieval_orchestrator.js
```

负责：

- 正式记忆只读读取；
- 行记录转换为召回项目；
- 按表选择注入内容；
- Prompt 记忆块组装；
- 检索运行态统计；
- 索引维护与召回预览。

主控制器只负责把当前角色、表格策略、保存函数和页面刷新回调注入该模块。

### 2. 审核应用编排

新模块：

```text
js/features/memory/review_orchestrator.js
```

负责：

- AI 更新结果解析；
- 字段级策略路由；
- 单表审核批次生成；
- 审核接受、拒绝、取消和回滚；
- 正式记忆变更回执；
- 运行态字段保存；
- XML 更新应用。

审核数据仍由原审核状态模块保存，正式写入仍通过 Domain 与 WriteGateway，不建立第二套数据层。

### 3. 记忆包编排

新模块：

```text
js/features/memory/package_orchestrator.js
```

负责：

- 单模板导出；
- 当前角色记忆包导出；
- 全部模板导出；
- 安全导入预演和导入执行；
- 结构、行、候选和运行态 ID 重映射；
- 跨角色导入后的运行状态清理。

底层 ID 映射规则继续复用 `package_adapter.js`。

## 领域所有权

三个新模块通过已有领域门面提供：

| 模块 | 所属领域门面 |
|---|---|
| `packageOrchestrator` | `memoryFoundationDomain` |
| `reviewOrchestrator` | `memoryGovernanceDomain` |
| `retrievalOrchestrator` | `memoryRetrievalDomain` |

主控制器仍只允许依赖七个领域门面和架构门禁，不允许直接 require 内部模块。

- 字段级写入策略；
- 审核操作；
- 候选和长期晋升；
- 角色覆盖设置；
- 记忆包格式；
- 正式记忆数据结构。
