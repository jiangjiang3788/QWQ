# 章鱼机 V2.14-R9：生命周期维护闭环、记忆来源变化链与 Proment 桌面修复

## 目标

R9 完成 V2.14 计划中的两项治理能力：

1. 生命周期健康、到期、复核、归档、冲突和人工合并形成可预演、可确认、可回滚的闭环；
2. 每条正式记忆统一展示来源、写入原因、首次与最近出现、字段变化、关系和合并记录。

同时修复 Proment 已存在但未出现在桌面应用网格的问题。

本版仍按单人自用设计：不增加多人锁、远程 revision、后台服务器或自动删除正文。

## 一、Proment 桌面入口修复

Proment 页面、图标和打开函数原本都存在，缺失点位于桌面应用清单：`homeAppIds` 没有包含 `proment`。

R9 将其恢复到桌面网格：

```text
世界书 · 剧场 · 收藏 · 提醒 · 搜索 · Proment · 外观 · 数据
```

点击后继续使用原入口：

```text
setupMagicRoomApp()
→ magic-room-screen
```

高级应用总开关仍然生效；关闭高级应用时，Proment、正则、思维链等高级入口可继续按原策略隐藏。

## 二、生命周期健康中心

管理页新增“生命周期与变化链”入口，展示：

- 健康分数；
- 当前有效、待确认、冲突、过期和归档数量；
- 来源分布；
- 缺少可验证来源的旧迁移记录；
- 30 天内到期记录；
- 到期或需要复核的记录；
- 冲突记录；
- 内容完全一致的可能重复组；
- 最近归档记录；
- 上次维护结果。

健康报告与维护预演只读取克隆快照，不修改正式记忆。

## 三、预演—确认—执行—回执

生命周期维护流程为：

```text
生成只读维护计划
→ 显示检查数和预计变化数
→ 用户确认
→ WriteGateway 完整快照事务
→ 应用状态变化
→ 持久化
→ 统一回执
```

执行前会明确显示预计：

- 过期多少条；
- 归档多少条；
- 转为待确认多少条。

维护不会：

- 删除记忆正文；
- 自动合并两条正式记忆；
- 自动覆盖冲突字段；
- 在普通聊天召回时偷偷执行。

保存失败时通过统一写入网关恢复完整 `memoryTables` 快照。

生命周期任务每次人工点击都会建立新的本地任务，不会被以前已经成功的维护任务错误去重。

## 四、生命周期状态规则

支持状态：

```text
当前有效
尚不确定
存在冲突
已被替代
已归档
已过期
```

自动维护只处理明确配置的条件：

- 到达 `expiresAt`：标记过期；
- 到达 `reviewAt` 且未经用户确认：标记待确认；
- 开启自动归档且超过未确认天数：标记归档。

没有设置到期、复核或自动归档规则的记录保持原状态。

## 五、来源与变化链

行检查器新增“来源与变化”页签。统一展示：

- 主要来源；
- 是否经用户确认；
- 来源数量；
- 稳定记录标识；
- 首次出现时间；
- 最近出现时间；
- Upsert 匹配次数；
- 最近变化事件；
- 来源消息、候选、审核批次或人工操作引用。

统一事件包括：

```text
创建记录
更新字段
匹配并更新原记录
补充来源证据
确认有效
延后复核
归档 / 恢复 / 过期 / 待确认
建立替代、冲突或相关关系
解除关系
合并记录
人工修改来源与时效
生命周期维护
旧版变化记录
```

事件只保留短摘要和引用标识，不复制完整聊天正文。

## 六、旧数据兼容

旧记录可能只有：

- `evidence.sourceRefs`；
- `versionLog`；
- `sourceMessageIds`。

来源页会只读派生旧来源和旧变化，不要求先修改正式记忆。

新变化首次写入时，旧来源可以迁入紧凑事件列表。修复了两个关键持久化断点：

1. 旧来源迁移后重新取得当前来源状态，避免创建事件写入失效对象引用；
2. 正式行规范化保留 `meta.provenance`，避免页面刷新或再次加载后变化链丢失。

事件支持 `eventKey` 幂等，重复操作不会反复生成同一条变化记录。

## 七、创建、更新、Upsert 与合并接入

正式 Domain 写入现已记录：

- 新建正式行：`create`；
- 字段实际变化：`update_field`；
- Upsert 命中原记录：`upsert_match`；
- 无变化写入：不产生字段变化事件。

人工确认、归档、延后复核和关系处理也会写入同一变化链。

人工合并时，保留行记录：

- 被合并行 ID；
- 复制字段；
- 来源合并数量；
- 合并原因。

被合并正文不会被后台维护自动删除。

## 八、迁移快照安全

Schema 升级为：

```text
3.0 → 3.1：补齐生命周期与来源变化链语义
```

迁移快照导入其他角色时：

- 变化链里的相关行 ID 跟随正式行 ID 重映射；
- 删除源角色消息和轮次引用；
- 清空旧事务 ID 与操作 ID；
- 保留人工引用和不依赖源聊天的短摘要；
- 完整备份继续保留原角色的全部来源和运行状态。

```text

```

## 十、本版边界

R9 不做：

- 自动删除旧迁移记忆；
- 自动合并疑似重复正文；
- 根据模型相似度自动覆盖正式档案；
- 浏览器真实点击与 IndexedDB E2E；
- 删除历史兼容字段；
- 自动加密备份。

这些分别留给 V2.15-R0 和可选的 V2.15-R1。
