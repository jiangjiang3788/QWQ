# 章鱼机 V2.11-R2：相关记忆侧栏与标签管理说明

## 1. 本版目标

V2.11-R2 延续 R0 的扁平表格和 R1 的模型标签、跨表整理能力，不再增加新的常驻卡片或外层工作台。

本版解决三个问题：

1. 一条记忆缺少统一的详情入口，标签、来源、关联和高级操作分散；
2. 用户无法直接查看跨表相关记忆，也无法明确建立关联、冲突或替代关系；
3. 标签虽然能够由模型生成，但缺少锁定、人工修正、同义标签合并和单条重算的完整管理链路。

## 2. 用户界面

### 2.1 单层记忆详情侧栏

行操作菜单新增“关联与标签”。打开后显示一个屏幕级侧栏：

- 当前表、行号和记忆摘要；
- 主题、场景、主体和作用标签；
- 标签锁定；
- 大模型重新生成标签；
- 跨表相关记忆；
- 相关度和选择依据；
- 建立关联、冲突和替代关系；
- 同义标签合并。

侧栏使用 `position: fixed`，不会进入表格布局，也不会把主表继续压窄。

桌面端固定在右侧；窄屏下占满视口。侧栏内部使用分隔线而不是卡片套卡片。

### 2.2 阅读与编辑边界

主表继续保持 R0 的阅读优先模式。标签和关系管理只在用户主动打开某条记录时加载，不为所有行常驻创建编辑器和按钮。

## 3. 相关记忆服务

新增：

```text
js/features/memory/relation_service.js
```

该服务扫描当前角色绑定模板中的全部行式记忆，并按以下信号计算关联：

- 主题标签重合；
- 场景标签重合；
- 主体标签重合；
- 记忆正文相似；
- 来源消息重合；
- 已存在的人工关系。

技术字段不会参与正文相似度，包括记录 ID、迁移状态、更新时间、置信度等，避免把迁移标记或内部元数据误判为相似记忆。

### 3.1 谨慎的重复与冲突判断

自动分析默认只提供“相关记忆”候选和相关度依据。

“冲突”只有在以下情况生效：

- 数据中已经存在明确冲突关系；
- 用户在侧栏中人工确认冲突。

“替代旧记录”同样需要用户明确操作。本版不会仅凭文本相似就自动删除、合并或覆盖记忆。

### 3.2 复用现有关系结构

关系继续保存到：

```js
row.meta.relations
```

复用既有 lifecycle 关系协议，不创建平行关系数据库。支持：

- `relatedTo`
- `conflictsWith`
- `supersedes`
- `supersededBy`

关系操作会双向维护并清除检索缓存。

## 4. 标签管理

### 4.1 继续复用 tagBundle

标签仍写入：

```js
row.meta.tagBundle = {
  topic: [],
  scene: [],
  entity: [],
  effect: ''
}
```

没有新增 `aiTags`、`generatedTags` 等平行字段。

### 4.2 标签锁定

锁定状态保存在：

```js
row.meta.tagLocked
```

锁定后：

- 后续整理任务不能用模型结果覆盖标签；
- 批量同义标签合并会跳过该记录；
- 用户仍可在详情侧栏中人工修改；
- 用户主动点击“大模型重算”时允许强制重算，但锁定状态本身不解除。

### 4.3 单条模型重算

新增专属 AI 操作：

```text
memory.tags.regenerate
```

任务名称：

```text
memory-table-tags
```

重算请求携带当前记忆和少量相关记忆，返回现有 `tagBundle` 协议。请求与结果会进入 AI 操作中心。

### 4.4 同义标签合并

用户可以在当前角色全部记忆表中，将某个主题、场景或主体标签统一替换为另一个标签。

操作特点：

- 只修改指定标签维度；
- 自动去重；
- 跳过锁定记录；
- 不修改记忆正文；
- 修改后清除旧检索向量和缓存。

## 5. 代码收敛

新增模块：

```text
js/features/memory/relation_service.js
js/features/memory/row_inspector.js
js/features/memory/row_inspector_controller.js
```

职责分别是：

- 关系分析和持久化；
- 详情侧栏纯渲染；
- 标签与关系操作控制。

主控制器只负责把当前角色、表格状态和保存函数传给侧栏控制器，不再继续堆入整套关系 UI 逻辑。

`memory_table.js` 当前约 4,518 行，保持低于 R1 专项门禁的 4,520 行。

## 6. 真实数据核验

使用清理优化版 `.ee` 中角色“阿沉”的实际数据：

- 行式记忆：209 条；
- 目标记录：稳定长期特征库第 99 行；
- 内容：不喜欢被用食物或礼物敷衍哄劝；
- 当前标签：长期档案条目、哄劝、日常聊天、soft_preference。

关联服务找到 8 条候选。最高相关记录为：

> 不喜欢道歉，不喜欢在生气时被用食物或礼物哄劝

相关度约 62%，依据为：

- 主题重合 100%；
- 场景重合 100%；
- 内容相似约 54%。

系统只把它显示为“相关记忆”，没有未经用户确认就自动合并或标记冲突。

## 8. 兼容边界

本版没有修改：

- IndexedDB schema；
- 模板和记忆行基础格式；
- `tagBundle` 和 `relations` 既有结构；
- 聊天消息格式；
- 结构化整理 Prompt 的跨表上下文协议；
- `.ee` 备份格式；
- 现有审核、任务、召回和回滚机制。

## 10. 下一阶段建议

V2.11-R3 应继续收敛，而不是增加新框：

1. 将关联候选加入整理审核，使模型可以提出“更新已有、合并、补证据或保持不变”；
2. 增加人工关系在跨表整理 Prompt 中的优先级；
3. 把模板设计器、导入导出和历史页面继续从 `memory_table.js` 拆出；
4. 增加标签质量统计和低质量标签批次，但放在现有管理页，不放回主表上方；
5. 使用真实 Provider 做一次标签重算和审核闭环验证。
