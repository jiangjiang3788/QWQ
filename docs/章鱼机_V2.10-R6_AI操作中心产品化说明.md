# 章鱼机 V2.10-R6：AI 操作中心产品化说明

## 1. 本版目标

V2.10-R6 将 R0—R5 已建立的操作、请求、Prompt 来源、父子任务和数据变化回执，整理成可以长期使用的产品界面。

本版不修改聊天生成、记忆提取、小剧场、识图或 Provider 请求逻辑，重点解决以下问题：

- 历史记录只能看最近几条，无法按功能、状态、角色或时间查找；
- 普通用户打开详情后会直接看到大量 Prompt 与 JSON；
- 复制操作记录时缺少稳定的脱敏报告格式；
- 操作记录虽然有条数上限，但缺少总容量预算；
- 密钥可能出现在对象字段之外的 URL、错误文本或字符串正文中。

## 2. 三层查看模式

操作中心新增三种查看级别，并将用户的选择保存在本地界面设置中。

### 普通模式

面向日常使用，只显示：

- 操作结果与状态；
- 执行阶段；
- 后台子任务；
- 数据变化；
- 记忆最终请求核验；
- 模型请求次数、模型、字符量和来源类别。

普通模式不展示原始 JSON，也不展开 Prompt 正文。

### 详细模式

用于核对一次请求实际使用了什么：

- 角色档案、世界书、结构化记忆、日记和向量记忆；
- 聊天历史、本次输入和输出规则；
- 每个来源的发送状态、证据等级、指纹和正文；
- 跳转到 Proment 或对应业务管理页。

详细模式不显示最终 Provider 原始请求体。

### 高级模式

用于开发与故障分析，在详细模式基础上增加：

- 脱敏后的最终请求体预览；
- 请求端点、HTTP 阶段和性能信息；
- 高级结果数据；
- 最终合并 Prompt 核对视图。

## 3. 操作历史工作台

操作中心首页新增历史筛选区域，可组合使用：

- 关键词：操作标题、类别、类型、阶段、摘要、角色名、模型、Provider、请求来源和错误；
- 状态：进行中、等待、成功、失败、取消、中断、跳过；
- 分类：聊天、记忆、小剧场、识别、图片生成、通话等；
- 具体功能：按 Operation type 精确筛选；
- 开始日期和结束日期。

默认显示 20 条，可逐步显示更多，最多读取当前会话保留的 100 条操作。

## 4. 脱敏报告

### 单次操作报告

操作详情可：

- 复制当前查看级别对应的 Markdown 报告；
- 下载当前操作及其后台子任务的脱敏报告。

### 筛选历史报告

历史页可以导出当前筛选结果，报告包含：

- 筛选条件；
- 操作摘要；
- 阶段、请求和数据变化；
- 当前操作历史存储状态；
- 按当前查看级别控制的来源正文或原始请求预览。

### 报告协议

新增只读导出协议：

```text
ovo.operation-report.v1
ovo.operation-history-report.v1
```

报告不会写回数据库，也不会改变原操作记录。

## 5. 脱敏范围

R6 在原有“敏感字段名隐藏”基础上，增加字符串级脱敏。以下内容即使出现在 URL、错误文本、Prompt 字符串或序列化 JSON 中，也会被替换：

- `Authorization: Bearer ...`；
- `api_key=...`、`key=...`、`token=...`、`access_token=...`；
- `secret=...`、`password=...`；
- JSON 字符串中的 API Key、Token、Authorization、Secret 和 Password；
- 常见 `sk-...` 和 `AIza...` 密钥形态。

脱敏同时应用于：

- 会话操作记录；
- 请求正文预览；
- 请求端点；
- 错误信息；
- 数据变化文本；
- 复制和下载报告。

## 6. 容量控制

操作记录继续使用 `sessionStorage`，不进入角色数据库、记忆数据库或普通 `.ee` 数据备份。

当前策略：

- 内存最多保留 100 条操作；
- 最近 8 条可以保留详细请求、Prompt 来源正文和变化前后内容；
- 更早记录自动只保留摘要、来源元数据和统计；
- 会话持久化总预算约 900,000 字符；
- 超过预算时依次压缩请求正文、Prompt 正文、结果和阶段细节；
- 必要时丢弃最旧的已完成记录，但不主动丢弃正在运行的记录；
- 操作中心显示当前占用、预算及是否发生自动压缩。

该策略用于避免操作中心本身成为新的备份或浏览器存储膨胀来源。

## 7. Operation Runtime 新增接口

```text
list(options)
getFacets(options)
getStorageStats()
buildOperationReport(id, options)
exportReport(id, options)
exportHistory(options)
redactSensitiveText(text)
```

`list(options)` 支持：

```text
query
status / statuses
category / categories
type / types
from
to
rootsOnly
limit
```

## 8. 兼容边界

本版未修改：

- IndexedDB schema；
- 角色、群聊和聊天消息格式；
- 结构化档案、日记和向量数据格式；
- 世界书格式；
- `.ee` 备份格式；
- Prompt 组装结果；
- Provider 请求次数和请求内容；
- R4 AI 能力目录；
- R5 Prompt 来源协议。

旧的 `ovo_operation_history_v1` 会话记录仍可读取；刷新前处于运行或排队状态的记录仍会标为“已中断”。

## 10. 自动验证

本版验证覆盖：

- 全量静态架构检查；
- 84 个生产 JavaScript 文件语法检查；
- 33 组 Node 自动回归；
- 查询、状态、分类、功能和时间筛选；
- 普通、详细、高级报告边界；
- URL、对象、错误文本和正文中的敏感数据脱敏；
- 大型操作历史的容量预算与自动压缩；
- R0—R5 操作中心历史回归；
- V2.1—V2.9-R11 记忆、设置、启动和导航历史回归。

## 11. 未声称完成的验证

当前环境没有使用用户的真实外部模型密钥，因此没有执行真实 Provider 线上端到端调用。

移动端触摸、系统下载权限和浏览器文件下载行为仍建议在实际部署地址复测。导出与筛选逻辑已通过运行时和静态自动检查，但没有将受环境限制的真实移动浏览器交互虚报为通过。
