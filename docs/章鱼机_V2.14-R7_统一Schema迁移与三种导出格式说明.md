# 章鱼机 V2.14-R7：统一 Schema 迁移与三种导出格式说明

## 目标

R7 解决两个长期混在一起的问题：

1. 旧记忆包由不同模块在读取时各自修补，缺少统一版本迁移顺序；
2. 原来的“导出模板”和“导出记忆包”没有清楚区分结构迁移、跨角色迁移和原位恢复。

本版不改变正式记忆表结构、字段写入策略、审核流程和召回逻辑。

## 一、统一 Schema 迁移注册表

新增模块：

```text
js/features/memory/schema_migrator.js
```

当前记忆包 Schema：

```text
3.0
```

旧 `2.8` 包按固定路径迁移：

```text
2.8
  ↓ 补齐包类型、导出语义和传输策略
2.9
  ↓ 统一三种包格式
3.0
```

每个迁移步骤都包含：

- 唯一步骤 ID；
- 来源版本；
- 目标版本；
- 中文变更说明；
- 纯函数迁移；
- 迁移追踪记录。

### 迁移安全

导入时先执行：

```text
解析 JSON
→ 识别 Schema
→ 查找完整迁移路径
→ 在深拷贝中预演
→ 校验目标格式
→ 用户确认
→ 执行导入或恢复
```

以下情况不会写入：

- JSON 无法解析；
- 不是结构化记忆包；
- 缺少必要数据；
- 没有连续迁移路径；
- 包版本高于当前支持版本；
- 完整备份对应的角色不一致。

源对象和源文件不会被迁移器修改。当前 `3.0` 包再次迁移时不会重复执行旧步骤。

## 二、三种导出格式

### 1. 模板

界面名称：

```text
导出模板
```

用途：复制表结构和默认策略。

包含：

- 模板；
- 表格；
- 字段；
- 表格职责；
- 采集、写入和召回默认策略。

不包含：

- 角色正式记忆；
- 字段锁定；
- 候选和审核；
- 游标与运行态；
- 索引和历史。

包标识：

```text
packageProfile: template_bundle
```

导入时为模板、表格和字段生成新 ID，避免和现有模板冲突。

### 2. 迁移快照

界面名称：

```text
迁移快照
```

用途：把正式记忆迁移到另一个角色或另一份本地数据。

包含：

- 模板结构；
- 正式记忆；
- 字段锁定；
- 可验证关系；
- Sidecar 候选；
- 质量测试配置；
- 全局运行设置的可迁移部分。

导入时：

- 重新生成模板、表格、字段、行和候选 ID；
- 重映射关系、晋升目标和候选目标；
- 清空源角色消息引用和处理游标；
- 清空待审核批次、任务历史和检索运行历史；
- 不携带可重建的行向量。

包标识：

```text
packageProfile: portable_snapshot
```

### 3. 完整备份

界面名称：

```text
完整备份
```

用途：恢复当前角色的完整结构化记忆状态。

包含：

- 原模板、表格、字段和行 ID；
- 全部正式记忆；
- 字段锁定；
- 短期候选；
- 审核批次；
- 任务、反馈和质量状态；
- 生命周期状态；
- 检索索引和使用统计；
- 角色运行态；
- 游标和更新历史。

包标识：

```text
packageProfile: full_backup
```

安全限制：

- 只允许恢复到导出时的同一个角色 ID；
- 跨角色导入会被拒绝，并提示改用迁移快照；
- 恢复通过 `WriteGateway` 串行执行；
- 保存失败时同时恢复角色记忆和模板快照。

## 三、用户界面

管理工具栏现在明确显示：

```text
导入
导出模板
迁移快照
完整备份
```

模板列表中单个模板提供：

```text
导出模板
迁移快照
```

完整备份始终按当前角色导出，不提供单表或单模板完整备份，避免产生语义不完整的“半备份”。

## 五、架构边界

新增 `schemaMigrator` 归属 `memoryFoundationDomain`。

```text
memoryFoundationDomain
├─ domain
├─ packageAdapter
├─ schemaMigrator
├─ packageOrchestrator
├─ writeCoordinator
└─ writeGateway
```

职责区分：

- `schemaMigrator`：只负责版本识别、迁移路径、预演和格式校验；
- `packageAdapter`：只负责跨角色导入时的 ID 重映射；
- `packageOrchestrator`：只负责导出、用户确认和导入/恢复编排；
- `WriteGateway`：负责完整备份恢复的持久化与失败回滚。
