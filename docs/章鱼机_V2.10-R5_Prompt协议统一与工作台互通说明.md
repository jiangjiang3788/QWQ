# 章鱼机 V2.10-R5：Prompt 协议统一与工作台互通

## 1. 本版目标

V2.10-R5 不重写聊天、小剧场、记忆、识图或日记的 Prompt 生成逻辑，而是在现有 `OVOPromptTrace` 上建立统一来源协议，使不同业务模块上报的来源具有一致含义，并让 AI 操作中心、Proment、世界书和记忆工作区能够互相定位。

本版原则：

- 不改变最终 Provider 请求体；
- 不新增平行 Prompt 系统；
- 不复制角色、聊天或记忆数据库；
- 旧 `promptSources` 调用可以继续工作，由追踪层统一规范化；
- UI 与底层协议同步落地。

## 2. 统一来源协议 v2

每个来源区块及来源条目现在统一为 `ovo.prompt-source.v2`，主要字段包括：

- `state`
  - `sent`：已实际发送；
  - `verified`：业务来源已与最终请求核对；
  - `contributed`：参与组装，但不能逐字符定位；
  - `excluded`：候选存在，但本次未发送。
- `evidence`：说明证据来自最终请求精确提取、最终请求精确推导、业务组装源精确上报或来源核对。
- `fingerprint`：正文及条目生成的 FNV-1a 内容指纹，用于辨认历史快照。
- `navigation`：对应的 Proment、世界书、结构化档案、回忆日记或向量记忆入口。
- `sourceId`：角色、世界书或其他业务来源 ID。
- 原有 `content`、`chars`、`count`、`reason`、`clipped`、`items` 和 `metadata`。

整个请求快照升级为 `ovo.prompt-trace.v2`，同时记录：

- `operationId`；
- `operationType`；
- 操作业务范围 `scope`；
- Provider、模型、任务和来源；
- 按状态及类型汇总；
- 可导航来源数量。

内容指纹只用于辨认记录，不参与 Prompt 排序、筛选、预算或模型请求。

## 3. 操作中心变化

每个发送内容来源现在显示：

- 统一状态；
- 证据说明；
- 内容指纹；
- 字符数量；
- “在 Proment 核对”；
- 对世界书、结构化档案、回忆日记和向量记忆提供“打开来源管理”。

旧 R1—R4 操作历史若没有 v2 字段，仍使用原有字段展示，不影响读取。

## 4. Proment 互通

操作中心打开 Proment 时，会通过受控的 `proment-focus.v1` 定位：

- operation；
- request；
- source；
- 角色范围。

Proment 优先展示该真实来源的：

- 操作与请求名称；
- 发送状态；
- 证据等级；
- 字符数；
- 内容指纹；
- 业务入口；
- 角色范围；
- 来源说明和实际正文。

这不是重新组装 Prompt，而是读取操作中心已经保存的同一次网络请求快照。

## 5. 来源管理跳转

- 世界书来源 → 世界书页面；
- 结构化记忆、角色档案实时状态 → 对应角色的结构化档案；
- 日记记忆 → 对应角色的回忆日记；
- 向量记忆 → 对应角色的向量记忆；
- 角色档案、用户档案、聊天、系统规则、任务规则等 → Proment。

进入角色相关页面前，只在能够找到对应角色时更新当前私聊角色，不会把不存在的 ID 写入角色数据。

## 7. 数据与兼容边界

本版没有修改：

- IndexedDB schema；
- 角色、群聊、消息格式；
- 结构化记忆、日记、向量数据结构；
- 世界书数据结构；
- `.ee` 备份格式；
- Provider 请求体和请求次数；
- Prompt 生成与注入决策。

操作记录仍受现有会话历史、正文截断和脱敏策略约束，不进入角色数据库或普通完整备份。
