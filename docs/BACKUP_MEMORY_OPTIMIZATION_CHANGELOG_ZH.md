# 备份与记忆冗余优化变更说明

## 已修复

- 修复任务结果持久化完整 `runtime/history` 导致任务队列递归膨胀的问题。
- 为持久化任务结果加入白名单、数量限制和 32 KB 上限。
- 已完成任务归档时移除 payload，仅保存紧凑结果。
- 不再给每条状态快照重复写入 `replacePattern`。
- 备份改为标准 UTF-8 字节写入，并对实际字节计算 SHA-256。
- 导入兼容旧 JSZip 产生的 CESU-8 代理对，但仍要求规范化后哈希完全匹配。
- 新增“精简备份（推荐）”和“原始完整备份”两种模式。
- GitHub 自动备份默认使用精简模式。
- 显式冗余清理会删除停用 `unifiedMemory`、重复状态模板和已完成任务过程快照。
- 默认记忆模板文件名与测试、构建工具统一。
- 发布清单由实际文件重新生成。

## 保留

- 聊天正文与消息数量。
- `memoryTables.data` 结构化记忆正文。
- 记忆回滚历史、反馈、质量、审核和 sidecar 数据。
- 待审核与排队任务。
- 全部当前 Dexie 数据表和非敏感 localStorage。

## 新增检查

- 旧 CESU-8 备份兼容测试。
- 超大递归任务结果持久化上限测试。
- 完整 V2.1–V2.9-R11 回归检查。
