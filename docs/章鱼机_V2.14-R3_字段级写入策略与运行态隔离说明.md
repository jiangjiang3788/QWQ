# 章鱼机 V2.14-R3：字段级写入策略与运行态隔离说明

## 一、版本定位

V2.14-R3 基于 R2 的正式写入门禁，继续解决记忆可信度问题：一张表可以允许自动更新，但表内并不是所有字段都适合直接成为正式用户记忆。

本版把写入判断从“只看表格”细化为“表格策略 + 字段策略”，同时保持单人自用的轻量设计，不建立复杂权限系统。

## 二、字段级策略

每个字段新增四项设置：

- 信息主体：用户、角色、双方关系、系统；
- 证据要求：用户明确表达、允许推断、仅人工；
- 字段写入：继承表格、直接生效、先确认、进入候选、仅运行态、仅人工；
- 最低置信度：0–100。

旧模板没有字段策略时，会根据表格层级、字段名称和分组安全推导。打开旧模板不会改动原对象；保存结构时才写入显式策略。

## 三、写入路由

自动更新解析出的每个字段都会附带：

```text
evidence = user_explicit | assistant_inferred
confidence = 0–100
```

系统再结合字段策略决定去向：

```text
直接写正式记忆
进入更新审核
进入候选
仅写会话运行态
阻止自动写入
```

用户明确事实可以直接进入短期正式记忆；推断性状态会降级为审核或候选；仅人工字段不会由模型自动填写。

## 四、避免半条记录

对于多行表新增记录，如果同一条新记录中有字段需要审核或候选，本版不会先写入部分字段形成半条记录，而是把整条记录作为一个待处理项提交。

已有记录的字段更新可以按字段分别路由：明确字段直接写入，推断字段进入审核或候选。

## 五、运行态隔离

角色判断、回应策略、边界提醒、下一步建议等字段可设为“仅运行态”。这些值保存在：

```text
chat.memoryTables.runtimeState.fieldValues
```

它们可以供当前会话和状态摘要使用，但不会写入正式 `memoryTables.data`，不会冒充用户档案，也不会出现在正式记忆变更历史中。

本版先完成数据和写入层隔离；“正式用户状态”和“角色运行态”在界面上彻底拆成两个区域，留到后续版本处理。

## 六、结构工作台

字段表增加中文列：

- 信息主体；
- 证据要求；
- 字段写入；
- 最低置信度。

日常界面继续隐藏内部字段 ID。旧字段会显示推导后的有效策略，用户可按需要覆盖。

## 七、AI 更新与 Sidecar

以下两条自动链统一使用字段策略：

- 周期整理 / 手动生成的 AI 表格更新；
- 跟随聊天回复运行的 Sidecar 更新。

Sidecar 的运行态值不会写正式档案；需要确认的字段会形成单表审核批次；候选字段会作为候选性质的待处理建议进入审核工作台。

```text

```

## 九、单人版边界

本版不实现：

- 多用户权限；
- 字段级账号授权；
- 服务端策略同步；
- 多设备策略冲突合并。

字段策略仅用于本地自动写入可信度和数据分层。
