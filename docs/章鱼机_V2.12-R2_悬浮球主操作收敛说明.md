# 章鱼机 V2.12-R2：悬浮球主操作收敛说明

## 1. 本版目标

V2.12-R2 只处理悬浮球操作中心的交互层级，不提前修改记忆模板、JSON 结构编辑器或记忆数据协议。

本版针对的问题是：

- API 与 Git 被放在独立二级页面；
- Proment 需要先进入状态页，再点击一次才能打开完整页面；
- 开发日志、功能覆盖和操作详情位于页面底部的卡片入口；
- 用户进入详情页后，需要依靠“返回”按钮重新寻找其他操作；
- 手机端操作入口容易只露出一部分。

产品目标是：

> 打开悬浮球后，全部主要操作立即位于页面顶部；内容区域只负责显示当前操作状态、历史或所选工具结果。

---

## 2. 最终交互结构

悬浮球打开后固定为三层：

```text
统一标题栏
├─ 当前页面名称
└─ 关闭

顶部主操作区
├─ 当前模型 / 刷新 / 应用
├─ 操作
├─ 详情
├─ Proment
├─ 日志
├─ 覆盖
├─ Git 上传
├─ Git 恢复
└─ Git 设置

内容区
└─ 当前操作、历史、日志、覆盖或操作详情
```

不再存在以下二级链路：

```text
API 与 Git → 模型切换 / Git 上传 / Git 下载 / Git 设置
Proment 状态 → 打开完整 Proment
操作中心底部 → 开发日志 / 功能覆盖 / 操作详情
```

---

## 3. 用户可见变化

### 3.1 顶部直接切换模型

顶部始终显示：

- 当前模型；
- 刷新模型列表；
- 应用所选模型。

不需要先进入“API 与 Git”。

### 3.2 八个主要操作始终可见

桌面端使用一行八项；手机端使用两行四列，避免水平翻页后才能发现后四个操作。

主要操作为：

1. 操作：当前状态和操作历史；
2. 详情：当前或最近一次操作的阶段、请求和数据变化；
3. Proment：直接打开完整 Proment；
4. 日志：查看页面运行日志；
5. 覆盖：查看 AI 能力登记与调用情况；
6. Git 上传：直接执行上传流程；
7. Git 恢复：直接执行下载恢复流程；
8. Git 设置：直接打开仓库配置。

### 3.3 工具内容不再拥有返回层级

进入日志、覆盖或操作详情后：

- 顶部主操作仍然保留；
- 可直接切换到另一项；
- 不再显示二级返回按钮；
- 关闭按钮始终位于统一标题栏。

### 3.4 高风险操作仍保留确认

“Git 恢复”虽然提升到顶部，但仍继续使用原有确认流程，不会因为扁平化而绕过安全确认。

## 4. 代码架构

新增纯展示模块：

```text
js/modules/quick_dock_action_bar.js
```

职责只有：

- 定义顶部主操作目录；
- 渲染模型控制区；
- 渲染主操作按钮；
- 根据当前内容标记选中状态；
- 在没有操作记录时禁用“详情”。

它不负责：

- API 请求；
- Git 上传或恢复；
- 页面跳转；
- 操作历史读写；
- 日志采集；
- Proment 数据。

这些行为仍由原有 `floating_ball.js` 和对应领域服务执行。

`floating_ball.js` 新增统一页面壳：

```text
renderPanelShell()
```

日志、覆盖、历史和详情复用同一个：

- 标题栏；
- 顶部主操作区；
- 内容挂载区域。

删除生产路径：

- `renderTools()`；
- `renderPromentStatus()`；
- `open-tools`；
- `open-proment`；
- 页面底部 `quick-dock-operation-tools` 操作卡片区。

主模块行数：

| 版本 | floating_ball.js |
|---|---:|
| V2.12-R1 | 1,061 行 |
| V2.12-R2 | 1,009 行 |
| 减少 | 52 行 |

新增的动作目录模块为 47 行。入口定义和页面行为已经分离，后续增加操作时不需要继续向主渲染函数复制卡片 HTML。

## 5. 数据与兼容性边界

本版没有修改：

- 操作历史协议；
- 操作报告协议；
- Prompt 来源协议；
- 记忆 schema；
- `.ee` 备份格式；
- API 配置结构；
- Git 配置结构；
- Proment 配置；
- AI 能力目录。

旧的 `QuickDock.open('main' | 'operation' | 'console' | 'coverage')` 继续可用。

`Proment` 顶部按钮直接复用原有完整 Proment 页面入口，不复制 Proment 状态数据。
