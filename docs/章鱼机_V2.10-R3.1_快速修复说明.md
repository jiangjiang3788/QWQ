# 章鱼机 V2.10-R3.1 快速修复说明

## 修复范围

本版本基于 V2.10-R3，只处理三项线上问题，不继续扩展 R4 功能。

### 1. 时间元数据被当成聊天气泡

此前聊天历史中的时间使用 `[消息时间：...]` 形式注入。该形式与应用的方括号消息协议过于接近，部分模型会在回复中复述，之后被消息解析器当成普通气泡。

本版本调整为：

- 历史消息时间使用 `<message_meta sent_at="..." />` 内部标签；
- 最终系统提示词增加 `message_metadata_protocol`，明确禁止输出时间元数据；
- 回复进入 Sidecar、状态栏和消息格式解析前，统一删除新旧时间元数据回显；
- 结构化记忆整理仍保留每条消息的完整日期、时间和时区，不影响时间理解。

### 2. 新消息偶尔不立即显示

数据库写入是成功的，但此前界面主要依靠每条消息的增量 DOM 追加。特殊消息分支、页面状态变化或异步执行时，增量追加可能漏掉，因此重新进入聊天后才显示。

本版本在一轮回复全部保存后增加一次当前聊天对账：

- 如果目标聊天仍然处于打开状态，执行完整 `renderMessages(false, true)`；
- 以已保存的聊天历史为准重建当前消息区；
- 不改变消息数据，也不会重复写入聊天记录。

### 3. 没看到角色档案记忆进入上下文

排查发现两个情况：

1. 用户自定义 Prompt 删除了 `{{共同回忆}}` 占位符时，结构化记忆/向量记忆/收藏日记虽然已经生成，但不会进入最终 system prompt；
2. 回复内档案系统的实时状态和待办已经存在于 `<memory_live_context>`，但操作中心没有把它单独识别为“角色档案记忆”来源。

本版本调整为：

- 自定义 Prompt 缺少 `{{共同回忆}}` 时，只要本次存在记忆内容，就自动补充 `<memoir>` 区块；
- 如果模板已经包含相同记忆内容，不重复注入；
- 操作中心从最终 system prompt 中提取 `<memory_live_context>`；
- 该内容以“角色档案记忆（实时状态与待办）”显示，并标记为“实际发送”；
- 结构化记忆、向量记忆和收藏日记仍按原记忆模式分别展示。

## 数据与兼容性

本版本没有修改：

- IndexedDB 表结构；
- 聊天消息数据格式；
- 结构化记忆数据格式；
- 角色档案数据格式；
- 备份文件格式；
- 模型请求次数。

消息区的完整刷新仅用于让界面与已保存数据保持一致，不会再次调用模型，也不会重复新增消息。

## 建议复测

1. 连续发送一次包含多条历史消息的聊天，确认角色回复中不再出现“消息时间”气泡；
2. 让模型一次返回多条气泡，确认无需退出聊天即可全部显示；
3. 使用删除了 `{{共同回忆}}` 的自定义 Prompt，确认操作中心仍显示结构化/档案记忆已实际发送；
4. 打开操作详情，核对“角色档案记忆（实时状态与待办）”与最终原始请求中的 `<memory_live_context>` 一致。
