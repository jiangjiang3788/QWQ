# 章鱼机 V2.14-R8：统一待处理协议与记录身份去重说明

## 目标

R8 解决两个长期存在但容易被界面掩盖的问题：

1. “待处理”只是把审核、候选、可靠性和任务临时拼在一起，各模块的状态、动作和计数规则并不一致；
2. 多行记忆缺少统一稳定身份，同一事件连续出现时容易被当成新记录重复新增。

本版面向单人本地使用，不引入复杂实体解析、云端去重服务或大型相似度模型。

## 一、统一 WorkItem 协议

新增模块：

```text
js/features/memory/work_item.js
```

以下来源现在统一转换为只读工作项：

- 更新审核批次；
- Sidecar 短期候选；
- 长期候选审核；
- 可靠性复核；
- 冲突复核；
- 失败和暂停的后台记忆任务；
- 待处理的召回反馈。

统一字段：

```text
id
类型与类别
来源引用
标题、原因和详情
风险等级
当前状态
创建时间
可用操作
是否允许批量选择
原始模块载荷
```

WorkItem 只负责描述与聚合，不取代各模块原来的正式存储。审核仍保存在审核模块，候选仍保存在候选模块，任务仍保存在任务队列。

### 待处理计数

待处理数量现在以 WorkItem 为唯一统计口径：

- 一条短期候选计为一个工作项；
- 一批单表更新审核计为一个工作项；
- 一条冲突或过期记忆计为一个工作项；
- 一个失败任务计为一个工作项；
- 普通日志、已完成历史和非操作性状态不计入待处理。

### 动作协议

每个工作项携带自己允许的动作，例如：

```text
进入审核
批准 / 拒绝长期候选
查看记忆
确认有效
30 天后复核
归档
打开短期候选
打开任务队列
清空待反馈
```

界面根据动作协议渲染，不再依赖大量按来源类型硬编码的按钮分支。

## 二、稳定记录身份

新增模块：

```text
js/features/memory/record_identity.js
```

正式多行记忆会获得：

```text
recordKey
sourceFingerprint
sourceFingerprints
sourceRefs
firstSeenAt
lastSeenAt
mergedFrom
matchCount
```

这些信息保存在行的 `meta.identity` 中，不增加用户可见字段，也不会改变原表结构。

### 记录匹配顺序

自动新增前按以下顺序寻找已有记录：

```text
显式 recordKey
→ 来源候选 ID
→ 事件 ID / 记录 ID / 任务 ID 等业务唯一字段
→ 来源指纹
→ 日期 + 标题
→ 内容指纹
```

匹配成功时执行更新，不新增第二行；没有匹配时才创建新记录。

### 旧数据兼容

旧记录第一次规范化时会自动补齐身份元数据。若旧表中已经存在标题和日期相同的多条独立记录，系统会为它们生成不同的唯一 `recordKey`，不会因补身份而自动合并或删除旧数据。

## 三、统一 Upsert

`Domain` 新增：

```text
upsertRow(...)
```

支持三种字段合并策略：

- `replace_non_empty`：新非空值替换旧值；
- `fill_empty`：只补旧记录中的空字段；
- `merge_non_empty`：文本和标签尽量合并，枚举、数字和日期使用新值。

已经接入：

- AI 自动新增多行记忆；
- 审核通过后的新增记录；
- Sidecar 候选保存到正式档案；
- Sidecar 候选合并；
- 其他通过正式 Domain 新增的自动记忆链。

长期候选晋升仍保留自身更严格的“来源行 ID、工作流目标和内容”幂等规则，并与统一记录身份兼容。

## 四、防止半条和错误回执

多行新增继续遵循 R3 的策略：只要一条新记录中包含需要审核或候选的字段，整条记录进入待处理，不先写半行。

R8 还修复了一个 Upsert 返回值断点：正式行在规范化后可能生成新对象引用。现在每次字段更新后都会重新读取正式行，确保：

- 正式数据；
- 操作回执；
- 审核结果；
- 界面显示

指向同一份最新记录。

## 五、实际记忆包验证

使用用户提供的修正版记忆包：

```text
模板：1
表格：8
字段：94
多行正式记忆：234
```

验证结果：

```text
已建立记录身份：234 / 234
重复 recordKey：0
真实待办表重复写入：33 行 → 33 行
匹配方式：业务唯一字段
实际变化：更新原记录 1 个字段
旧版候选 WorkItem：4 项
源记忆包 SHA-256：前后一致
```

## 六、本版边界

R8 不做：

- 自动删除历史重复记录；
- 模型驱动的复杂语义聚类；
- 跨角色自动合并；
- 生命周期批量归档；
- 来源和变化链的完整可视化。

这些内容分别留给 R9 的生命周期与来源链闭环。
