# 章鱼机 V2.13-R5.4：更新审核事务闭环说明

## 本版目标

本版延续 V2.13-R5.3 的每表策略收敛，完成更新审核链的事务闭环，并将连续三次回车触发改为完全静默。

## 一、连续三次回车静默触发

- 空输入框在 3 秒内连续独立敲击三次 Enter，触发一次 AI 回复。
- 30 秒冷却保持不变。
- 长按、组合键、输入框有文字、输入法合成状态均不会误触发。
- 成功、冷却中、AI 正在回复和触发失败均不显示 Toast；失败只写入控制台。

## 二、审核按记忆记录分组

- KV 表的全部字段归为一条记忆。
- 多行表按 rowId 分组；同一行修改多个字段只显示为一条记忆。
- 新增行作为一条独立记忆。
- 审核顶部同时显示“记忆条数”和“字段变化数”。
- 每条记忆支持：接受整条、拒绝整条、逐字段调整。
- 审核界面不显示内部 rowId。

## 三、一个审核批次只对应一张表

- 同一次 API 返回多张表的更新时，按 templateId + tableId 拆分为多个批次。
- 每个批次拥有独立待审核状态、游标、完成状态和回滚信息。
- 相关表只作为只读参考，不再混入同一写入批次。

## 四、每表写入策略决定审核

- direct：直接写入。
- review / promotion / candidate：进入该表自己的审核批次。
- manual_only：不进入模型提示目标，也不接受模型写入。
- 不再根据主聊天 API 或总结 API 决定是否审核。

## 五、审核事务原子化

应用、拒绝、取消和安全回滚均在完整 memoryTables 快照上执行：

1. 保存操作前快照；
2. 修改正式档案、审核队列、游标、任务和历史；
3. 持久化角色；
4. 持久化失败则恢复完整快照；
5. 尝试再次持久化恢复后的状态。

因此不会出现“档案已经改了但审核仍待处理”或“游标推进但草案仍存在”的半完成状态。

## 六、拒绝和取消语义

- 拒绝并跳过这段消息：不写入档案，推进该表游标，该范围不再自动整理。
- 取消本次整理，保留处理范围：不写入档案，不推进游标，之后可重新生成同一范围。
- 完成历史分别记录 rejected_skipped 和 cancelled_preserved。

## 七、操作回执

审核保存回执按记录和字段双计数：

- 已保存 1 条记忆（14 个字段）
- 不再显示“保存 14 项”造成 14 条记忆的误解。
