# 章鱼机 V2.11-R3.1：记忆侧栏重做说明

## 一、为什么重做 R3

V2.11-R3 的去重、合并、冲突和标签词表能力已经形成完整数据闭环，但重新检查真实备份截图后，界面没有达到继续扩展的标准：

- 去重审核、标签表单、相关候选和词表同时纵向堆叠在一个侧栏中；
- 桌面侧栏一次渲染 284 个内部节点、33 个按钮和 8 个输入控件；
- 手机端仍需要承载横向比较表，存在内容截断和操作拥挤；
- 用户进入“审核”后，下面仍保留不相关的标签、词表和候选区域，注意力不集中；
- 继续增加能力会再次形成“一个侧栏套所有功能”的新单体界面。

因此本版没有继续增加业务功能，而是把 R3 重做为 V2.11-R3.1。

## 二、重做目标

1. 保留 R3 的全部去重、合并、冲突、关系和词表能力。
2. 同一时刻只渲染用户正在处理的一类内容。
3. 审核时只显示比较和决策，不把标签、词表继续堆在下方。
4. 手机端不发生横向溢出，也不依赖缩小字体塞下桌面表格。
5. 不增加新的页面外壳、卡片容器或平行数据结构。

## 三、界面结构

### 3.1 普通详情模式

侧栏顶部只保留三项页签：

- 关联
- 标签
- 词表

三个区域互斥渲染，而不是通过 CSS 隐藏三套完整 DOM。

默认进入“关联”：

- 当前记忆摘要；
- 关系数量；
- 最相关的前 8 条候选；
- 审核、查看和仅关联操作。

“标签”页只渲染原有 `row.meta.tagBundle` 编辑器和模型重算入口。

“词表”页只渲染同义标签归一规则。

### 3.2 聚焦审核模式

点击候选“审核”后，原侧栏直接切换为聚焦审核，不再显示页签、标签表单、词表和其他候选。

审核模式只包含：

- 当前记录；
- 候选记录；
- 有差异的业务字段；
- 合并边界说明；
- 四个明确决策。

决策区固定在侧栏底部：

- 合并到当前；
- 合并到候选；
- 标记冲突；
- 仅建立关联。

### 3.3 手机端

390px 视口下：

- 侧栏使用 `left: 0; right: 0; width: auto`，不再因 `100vw` 与浏览器滚动条产生偏移；
- 字段比较从三列表格改为纵向字段对照；
- 决策按钮固定为两列；
- 页面测量结果为 `documentWidth = viewportWidth = 390`；
- 侧栏左右边界为 `0 / 390`，没有横向溢出。

## 四、代码调整

### 4.1 `row_inspector.js`

重写为几个纯渲染单元：

- `relationSection()`
- `tagSection()`
- `vocabularySection()`
- `reviewPanel()`
- `tabNavigation()`

普通模式只调用当前页签对应的渲染器；审核模式只调用 `reviewPanel()`。

### 4.2 `row_inspector_controller.js`

新增统一动作：

```text
switch-row-inspector-tab
```

状态只新增一个紧凑字段：

```js
inspectorTab: 'relations'
```

打开其他记录、取消审核或关闭侧栏时，会恢复到关联页，不保留错误上下文。

### 4.3 `memory_table.js`

主控制器仅负责保存和传递 `inspectorTab`，没有把页签渲染、审核布局和词表逻辑重新收回主文件。

当前 `memory_table.js` 仍为 4,518 行，没有因本次重做继续膨胀。

### 4.4 CSS

删除 R2/R3 重复叠加的侧栏和横向审核表样式，改成一套 R3.1 样式。

旧路径已移除：

```text
.memory-row-review-table-wrap
.memory-row-review-table
```

新审核字段使用：

```text
.memory-row-review-diffs
.memory-row-review-diff
```

## 九、判断

R3 的数据能力可以保留，但原界面不应继续扩展。R3.1 已把该问题重做并达到继续开发的门槛。后续版本应继续遵守：

> 一个侧栏只承载一个当前任务；不同能力通过页签或状态切换复用同一平面，不通过纵向追加区块扩展页面。
