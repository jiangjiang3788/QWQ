# 章鱼机 V2.12-R0：记忆表格交互统一第一版

## 1. 本版定位

V2.12-R0 以 V2.11-R7 为基线，只实施已确认规划的第一阶段：

1. 字段分组提升为表格一级结构；
2. 移除常驻“编辑”按钮；
3. 电脑和手机统一使用“单击选中、长按编辑”；
4. 键盘提供 Enter 编辑、Esc 退出；
5. 行式记忆表将标签提升为正式字段列；
6. 保持虚拟滚动、增量保存、撤销、侧栏、治理和备份兼容。

本版没有提前加入“记忆引用与作用”、悬浮球操作重构和表结构编辑器，避免不同领域再次耦合。

---

## 2. 用户交互变化

### 2.1 键值档案：字段分组提升一级

原来的字段附属分组标识被移除。表格现在按照模板字段的 `group` 配置，先生成一级分组，再显示组内字段：

```text
核心确认
├─ user_人格底色
├─ char_回应原则
├─ char_边界与底线
├─ 双方_核心关系定义
├─ 双方_称呼系统
└─ 双方_相处公约
```

没有分组配置的字段统一进入“其他字段”，不会在每个字段下面重复显示分组文字。

### 2.2 行式记忆：标签成为正式列

行表现在统一包含：

```text
记录｜标签｜字段组……
```

标签列直接读取现有 `row.meta.tagBundle` 和兼容标签数据，没有增加新的标签字段。标签可以作为搜索和筛选依据，序号与行操作仍保留在“记录”列。

### 2.3 统一编辑手势

桌面端和手机端使用同一套状态机：

| 操作 | 结果 |
|---|---|
| 单击/单点 | 选中当前字段或记录，不进入编辑 |
| 长按约 0.5 秒 | 进入编辑；再次长按完成编辑 |
| Enter | 对当前选中目标进入或完成编辑 |
| Esc | 退出当前编辑 |
| `⋯` | 打开非编辑类行操作菜单 |

“编辑字段”“编辑此行”等常驻按钮已经从生产 UI 删除。新增行后仍会自动进入新行编辑，避免降低录入效率。

## 3. 表头与字段组结构

行式表格使用双层表头：

```text
记录 | 标签 |              主要信息
             创建时间 | 类型 | 标题 | 内容 | ……
```

字段组来自模板 schema，不在视图层复制另一份配置。字段顺序仍以模板列顺序为准，同一组内不会被重新排序。

在窄屏设备上，表格只在自身滚动容器内横向滚动，页面宽度不会被撑开。

---

## 4. 代码分层

### 新增 `table_grouping.js`

职责：

- 按模板字段 `group` 生成稳定字段组；
- 保留字段原始顺序；
- 为键值字段生成稳定路径；
- 为未分组字段提供统一回退组。

它只处理结构，不接触 DOM 和持久化。

### 新增 `table_gesture.js`

职责：

- 统一 pointer 与键盘交互；
- 区分选中状态和编辑状态；
- 管理 480ms 长按计时；
- 忽略按钮、输入框、链接等交互元素；
- 调用既有局部表格刷新，不重建整个页面。

### 调整既有模块

| 模块 | 本版职责变化 |
|---|---|
| `table_session.js` | 增加选中字段/记录状态，编辑状态与选中状态统一 |
| `table_presenter.js` | 生成字段组、标签检索文本和表头模型 |
| `table_grid.js` | 渲染一级字段组、双层表头、正式标签列和手势目标 |
| `table_view.js` | 统一标签字段和记录状态展示 |
| `row_command_menu.js` | 删除“编辑此行”，只保留真正的行命令 |
| `table_workspace.js` | 提供统一交互提示和手势上下文 |
| `memory_table.js` | 只接入统一网格绑定，不复制长按业务逻辑 |

### 重复绑定修复

真实浏览器核验发现：单击触发局部刷新后，如果在新网格根节点重复绑定完整手势，长按会冒泡触发两次，表现为“进入编辑后立即退出”。

最终实现中，局部刷新只重新绑定虚拟滚动；完整手势只绑定在稳定工作区根节点。专项测试已经加入门禁，防止该问题回归。
