# 章鱼机 V2.13-R4 Sidecar 候选闭环说明

## 范围

本版本只完成短期 Sidecar 候选的业务闭环，不改造长期候选事务、全系统写入口或运行态分层。

## 主要变化

- “标记已整理”替换为“保存到档案 / 合并到已有记录 / 暂时忽略 / 删除”。
- 新增 `sidecar_candidate_service.js`，正式行新增与字段合并统一调用现有 `Domain`。
- 新增 `sidecar_candidate_controller.js`，由主记忆控制器注入聊天、保存、渲染、确认和错误处理上下文。
- 保存失败时恢复正式记忆数据和 Sidecar 候选快照，避免半完成状态。
- 旧 `processed` 且没有目标行的候选迁移为 `legacy_unverified`，不会被误判为已保存。
- 保存或合并后记录目标模板、目标表、目标行、操作 ID 和处理时间。
- 候选保存、合并具备幂等保护，重复点击不会重复新增或重复拼接。
- 正式行元数据保留 `sourceCandidateId` 与 `sourceRoundId`。

## 候选状态

- `pending`：待处理
- `promoted`：已保存到档案
- `merged`：已合并到档案
- `dismissed`：已忽略
- `deleted`：已删除
- `legacy_unverified`：旧版去向未验证
