# 章鱼机 V2.11-R5：大表虚拟化与按需行操作说明

## 1. 本版目标

V2.11-R5 不新增记忆业务功能，专门解决 R4 在真实 179 行长期记忆表中暴露的前端性能和维护性问题：

- 一次性渲染全部记录；
- 每行常驻一个包含 9 个动作的 `select`；
- 179 行会生成 179 个操作控件和 1790 个 `option`；
- 大量不可见 DOM 仍参与布局、样式计算和事件处理；
- 表格交互继续散落在 4000 多行主控制器中。

本版坚持此前确定的 UI 原则：表格仍是主体，不增加新的常驻卡片或外框。

## 2. 大表虚拟化

新增：

```text
js/features/memory/table_viewport.js
```

虚拟窗口默认参数：

```text
启用阈值：80 行
阅读行高：88px
上下缓冲：5 行
```

大表只渲染当前可见区域及少量缓冲行，并使用顶部、底部占位行保持正确滚动高度。滚动事件通过 `requestAnimationFrame` 合并，同一帧只更新一次可见窗口。

虚拟滚动状态按以下维度隔离：

- 角色；
- 模板；
- 表格；
- 生命周期筛选；
- 标签筛选；
- 搜索词；
- 阅读/结构模式。

切换表格或筛选不会把不同视图的滚动位置混在一起。重新渲染、打开侧栏或进入某一行编辑时，会恢复当前表格的窗口位置。

### 安全边界

以下情况不启用固定行高虚拟化：

- 少于 80 行的小表；
- JSON 完整结构模式；
- 审核层表格。

这些页面包含高度变化较大的编辑器、结构信息或审核动作，保持完整渲染可以避免错误估算行高。普通长期记忆阅读表使用虚拟窗口。

## 3. 按需行操作

R4 每行使用一个完整的 `select`，九个命令都常驻 DOM。R5 改为：

```text
每个可见行：一个轻量“⋯”按钮
整个页面：最多一个共享操作菜单
```

新增：

```text
js/features/memory/row_command_menu.js
```

共享菜单仅在用户点击某行时创建，支持：

- 关联与标签；
- 编辑此行；
- 高级使用策略；
- 来源与时效；
- 固定；
- 暂停使用；
- 上移；
- 下移；
- 删除。

点击菜单外部或按 `Esc` 会关闭。手机端菜单固定为底部两列布局，不压缩表格，也不撑宽页面。

## 4. 表格交互继续拆分

新增：

```text
js/features/memory/table_interaction.js
```

它统一处理：

- 行菜单打开；
- 行编辑开始/完成；
- 键值字段编辑开始/完成；
- 生命周期筛选；
- 标签筛选。

`memory_table.js` 只保留页面编排和业务动作入口。主控制器由 R4 的 4416 行降至 4393 行；本版增加三项性能模块后，主文件仍减少 23 行。

## 5. 真实数据运行结果

使用用户提供的清理优化版 `.ee`：

- 角色：阿沉；
- 聊天消息：19133 条；
- 模板：8 张表；
- 稳定长期特征：179 行。

### R4 与 R5 对比

| 指标 | R4 | R5 桌面 | 变化 |
|---|---:|---:|---:|
| 实际数据行 | 179 | 179 | 数据完整保留 |
| 当前 DOM 中的数据行 | 179 | 18 | -89.9% |
| 行操作控件 | 179 | 18 | -89.9% |
| 常驻行操作 `option` | 1790 | 0 | -100% |
| 输入/选择控件 | 180 | 1 | -99.4% |
| 页面 DOM 节点 | 5913 | 499 | -91.6% |

390px 手机端实际渲染 17 行；打开操作菜单时只增加一个菜单和 9 个按钮。页面宽度保持 390px，没有页面级横向溢出。

### 滚动后编辑核验

在滚动到长期表中段后：

- 目标行仍在虚拟窗口内；
- 只出现 1 个编辑行；
- 当前窗口仍为 18 行；
- 完成编辑后编辑状态归零；
- 不会恢复 1790 个行操作选项。

## 6. 兼容性

没有修改：

- 记忆模板 schema；
- `.ee` 备份格式；
- `row.meta`；
- `tagBundle`；
- 生命周期字段；
- 关系字段；
- 模型整理协议；
- 检索和审核数据。

筛选、编辑、关联侧栏、去重审核、长期候选晋升及历史恢复继续使用原有数据和动作。

## 8. 下一阶段建议

下一阶段不应继续增加表格功能框。更有价值的方向是：

1. 把模板设计器、导入导出和历史渲染继续移出 `memory_table.js`；
2. 为宽表增加可配置列冻结与列显示方案，而不是继续增加横向字段；
3. 建立浏览器性能门禁，例如 500 行表的最大 DOM、首屏渲染时间和滚动帧稳定性。
