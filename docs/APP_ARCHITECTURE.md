# 多 App 信息架构（V2.9-R9）

章鱼机首页采用手机桌面式单层网格。首页只展示彼此独立的工具，不再按“角色与聊天 / 创作 / 记录与工具 / 系统”分组。

## 首页应用

- 世界书
- 剧场
- 收藏
- 提醒
- 搜索
- 外观
- 数据

## 常用 Dock

Dock 是四个高频入口的唯一位置：

- 聊天
- API
- 记忆
- 设置

Dock 顺序仅由 `js/app_registry.js` 的 `dockAppIds` 维护。Dock 应用不得再次出现在首页网格中。

## 聊天与角色边界

- 不再提供独立“角色 App”。
- 不再提供独立“联系人 App”。
- 不再提供旧的聊天 / 联系人 / Apps 全局底栏。
- 用户从 Dock 的“聊天”进入聊天列表，在聊天列表右上角创建角色。
- 当前角色设置仍从聊天详情或“设置 → 角色设置”进入。
- 角色数据、聊天数据和用户档案数据没有删除。

## 页面导航

- `js/core/navigation_runtime.js` 是页面历史栈的唯一实现。
- `js/ui.js` 负责实际激活或隐藏 `.screen`。
- `window.switchScreen()` 是所有旧模块可调用的显式导航门面。
- `.back-btn[data-target]` 由主入口统一处理：优先回到真实上一级，没有历史时才使用 `data-target`。
- 页面模块不得再各自维护一套返回栈。

## 桌面和手机布局

桌面端与手机端使用同一套四列网格：

- 网格最大宽度 520px；
- Dock 最大宽度 500px；
- 大屏只增加留白，不拉伸图标间距；
- 不再显示分区标题和工作台式长列表。

## 约束

1. App 元数据、首页顺序和 Dock 顺序只在 `js/app_registry.js` 维护。
2. Dock 应用不得重复出现在首页。
3. 新页面必须通过 `switchScreen()` 或 `OvoNavigation.open()` 打开。
4. 返回按钮必须使用 `.back-btn[data-target]`，其中 `data-target` 只作为无历史时的兜底。
5. 删除入口不等于删除数据模型；角色、聊天和档案仍由原数据层维护。

## V2.9-R10：角色上下文与搜索导航

- 角色级 App 不得假设首页仍保留 `currentChatId`。
- 角色选择器必须通过能力检测兼容没有完整 `HTMLDialogElement` API 的 WebView。
- 搜索是独立首页 App，由 `setupSearchSystem` 初始化，不再依赖旧 Apps 页面。
- 搜索、收藏、记忆等首页 App 的无历史返回兜底统一为 `home-screen`。

## V2.9-R11：记忆工作区状态机

角色记忆页不再由 getter 持续覆盖 UI。首次进入角色时从 runtime 水合；之后所有显式导航通过统一提交函数同步 UI 与角色 runtime。这样导航状态、角色上下文和记忆工作区状态的职责保持分离。
