# 多 App 信息架构

V2.9-R6 保留手机桌面式多 App 体验，但执行“单一入口”原则。

## 首页 Dock

- 聊天
- 角色
- 记忆
- 设置

这四项是全局最高频入口，只在 Dock 出现，不再重复到桌面 App 网格。

## 桌面 App

- 世界书
- 剧场
- 收藏
- 提醒
- 搜索

桌面使用横向分页和原生 Scroll Snap，不使用长纵向列表。

## 上下文入口

- 联系人属于聊天域，保留在聊天底栏；
- API、数据和外观属于设置域，由设置 App 打开；
- 角色的聊天、记忆和设置由角色 App 的同一张角色卡打开。

## 高级工具

Proment、正则、思维链和状态栏继续由功能开关控制，默认从设置或明确的高级入口访问，不占用普通用户桌面。

## 约束

1. App 注册信息只在 `js/app_registry.js` 维护。
2. 首页不手写应用 HTML。
3. 页面切换和返回只通过 `OvoNavigation`。
4. 一个用户任务只保留一个主入口。
5. 设置只承载配置，不复制日常操作入口。
6. 新 App 必须声明归属：dock、main、settings、context 或 advanced。
