# 章鱼机 V2.11-R6：增量表格与状态收敛说明

## 一、本版目标

V2.11-R6 不扩展新的记忆功能，不增加新的卡片、页面层级或侧栏。

本版只处理 V2.11-R5 之后仍然存在的三个维护性问题：

1. 表格筛选、搜索、进入编辑等轻量操作仍会重建整个记忆工作区；
2. 表格数据模型、工作区渲染、虚拟窗口和控制器状态边界仍不够清晰；
3. 每次字段保存都会调用整页 `renderMemoryTableScreen()`，导致左侧表清单、侧栏和其他工作区节点无意义重建。

## 二、局部刷新

记忆表格主体新增稳定挂载点：

```html
<div data-memory-table-grid></div>
```

以下操作现在只替换该挂载点：

- 进入或完成行编辑；
- 进入或完成键值字段编辑；
- 生命周期筛选；
- 标签筛选；
- 表格内容搜索。

不会重建：

- 页面顶栏；
- 记忆、待处理、管理导航；
- 左侧八张表清单；
- 当前打开的关系侧栏；
- 自动整理抽屉；
- 其他工作区 DOM。

## 三、字段增量保存

字段值修改后：

1. 继续使用现有 `Domain.updateRowFieldValue()` 或 `Domain.setFieldValue()`；
2. 继续调用原有 `saveCharacter()`；
3. 将规范化后的值同步回当前编辑器；
4. 显示短暂的“已保存”状态；
5. 不再调用整页 `renderMemoryTableScreen()`。

保存状态使用轻量文本和 `aria-live`，没有增加卡片或弹窗。

完成编辑时，才局部刷新当前表格，将编辑器恢复为阅读单元格。

## 四、状态与渲染职责拆分

新增四个模块：

### `table_session.js`

统一处理表格会话状态：

- 当前表；
- 行编辑；
- 字段编辑；
- 生命周期筛选；
- 标签筛选；
- 搜索；
- 普通与 JSON 模式。

并确保行编辑和字段编辑互斥。

### `table_presenter.js`

负责把真实模板和数据转换为渲染模型：

- 可见字段；
- 搜索结果；
- 筛选结果；
- 原始行索引；
- 审核表信息；
- 虚拟窗口计划。

渲染器不再同时负责数据筛选和模型计算。

### `table_reconciler.js`

负责局部 DOM 更新：

- 保存虚拟滚动位置；
- 保存当前焦点和文本选择范围；
- 替换当前表格区域；
- 重新绑定虚拟滚动；
- 恢复焦点；
- 显示保存反馈。

### `table_workspace.js`

负责：

- 已绑定表格清单；
- 当前表解析；
- 左侧表清单；
- 表头和策略摘要；
- 表格挂载点；
- 工作区与表格渲染器的连接。

这些内容已从 `memory_table.js` 移出。
