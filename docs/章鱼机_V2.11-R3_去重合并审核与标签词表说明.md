# 章鱼机 V2.11-R3：去重合并审核与统一标签词表

## 1. 本版目标

V2.11-R3 延续 R0 的扁平表格、R1 的模型标签与跨表上下文、R2 的单层相关记忆侧栏。本版不增加新的常驻页面或外层框，而是在同一个侧栏中完成：

- 可能重复记忆的对比审核；
- 冲突、替代和关联决策；
- 安全的证据合并；
- 统一标签词表；
- 后续模型标签自动归一。

## 2. 去重与冲突审核

相关记忆不再只能“查看、关联、冲突、替代”。当候选被识别为可能重复、需要核对，或相关度达到 55% 时，可以先进入对比审核。

审核层仍位于原来的右侧详情侧栏中，显示：

- 当前记录与候选记录摘要；
- 真正有差异的业务字段；
- 可补齐的空字段数量；
- 双方都有内容的字段数量；
- 合并后的来源证据数量；
- 当前记录和候选记录的保留选择。

用户可以选择：

1. 保留当前记录并合并证据；
2. 保留候选记录并合并证据；
3. 保留两条并标记冲突；
4. 仅建立关联。

系统不会因相似度自动删除或自动合并。

## 3. 安全合并规则

新增独立模块：

```text
js/features/memory/merge_review_service.js
```

合并遵循以下边界：

- 只把候选中的值补入保留记录的空字段；
- 双方都有内容的字段绝不自动覆盖；
- 合并 topic、scene、entity 标签；
- 合并 sourceMessageIds 和证据引用；
- 合并已有相关关系；
- 使用现有生命周期模块将另一条标记为 `superseded`；
- 不物理删除另一条记忆；
- 清除保留记录的旧检索向量和缓存。

原始记录 ID、迁移状态、创建时间、置信度等技术字段不会进入差异对比，也不会被用于字段合并。

## 4. 合并审计与数据体积

每次审核结果只在：

```js
chat.memoryTables.mergeAudit
```

保留最多 40 条紧凑审计。审计只保存：

- 两条记录 ID；
- 所属表 ID；
- 采用的审核动作；
- 补齐的字段名；
- 合并来源数量；
- 标签是否变化；
- 内容差异数量。

不会保存完整记忆正文、完整角色、聊天历史、Prompt 或运行对象，因此不会重新制造递归大数据。

审核动作同时登记为操作中心事件：

```text
memory.merge.review
```

操作中心只记录审核类型、涉及的记录 ID、补齐字段数量和变化摘要，不复制两条记忆正文。

## 5. 统一标签词表

新增独立模块：

```text
js/features/memory/tag_vocabulary.js
```

词表保存在当前角色：

```js
chat.memoryTables.tagVocabulary
```

结构是紧凑的三维别名映射：

```text
topic
scene
entity
```

例如：

```text
坦白表达 → 表达
睡眠问题 → 睡眠
```

在侧栏中合并同义标签时，会同时：

1. 批量替换当前角色已有记忆中的旧标签；
2. 跳过已锁定标签的记忆；
3. 把旧标签与统一标签的关系写入词表；
4. 以后模型再次输出旧标签时自动映射到统一标签。

词表规则可以单独删除；删除词表规则不会反向修改现有记忆。

## 6. 模型整理与标签重算

`tag_service.js` 和 `update_service.js` 已接入词表：

- 自动整理新增或更新记忆时，模型标签写入前会经过词表归一；
- 单条“大模型重算”也会读取当前角色词表；
- 整理 Prompt 会告诉模型优先使用统一标签；
- 标签仍写入原有 `row.meta.tagBundle`；
- 没有新增 `aiTags`、`generatedTags` 等平行字段。

## 8. 真实数据核验

使用清理优化版 `.ee` 的实际数据副本：

- 角色：阿沉；
- 聊天消息：19,133 条；
- 行式记忆：209 条。

在稳定长期特征库中找到一组相关度 63.8% 的记录：

- 主题重合 100%；
- 场景重合 100%；
- 内容相似 58%；
- 系统分类为“需要核对”。

实际副本合并结果：

- 技术字段没有进入差异表；
- 只有“内容”被标记为双方都有内容；
- 保留记录正文没有被覆盖；
- 另一条记录进入 `superseded`；
- 紧凑审计不包含完整正文；
- 词表规则“坦白表达 → 表达”可被后续标签写入使用。

原始 `.ee` 文件未被修改。

## 9. 兼容边界

本版没有修改：

- IndexedDB 表结构；
- `.ee` 备份容器格式；
- 记忆模板结构；
- 记忆行字段结构；
- `row.meta.tagBundle` 和 `row.meta.relations` 的既有含义；
- 聊天 Prompt 的档案注入规则；
- 现有审核、任务、召回和生命周期接口。

新增的 `tagVocabulary` 与 `mergeAudit` 都是角色 `memoryTables` 下的紧凑可选字段，旧数据无需迁移即可使用。
