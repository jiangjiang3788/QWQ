# 章鱼机 V2.14-R5：有效策略解释器与设置来源可视化说明

## 一、版本定位

V2.14-R5 基于 R4 的纯读取召回，继续收敛记忆设置的实际执行规则。

此前系统已经将采集、写入、周期与召回配置放进统一表结构工作台，但同一张表仍可能同时受到模板策略、当前角色运行状态和全局周期默认值影响。用户可以看到多个设置，却不容易确认最终由哪一层生效。

本版新增唯一的有效策略解释器，让界面展示和业务执行共用同一套计算结果。

## 二、统一策略层级

每张表的最终策略按以下顺序计算：

```text
系统默认
→ 模板默认
→ 当前角色覆盖
→ 全局周期默认（仅当频率来源选择“全局”）
```

其中：

- 系统默认用于模板缺少显式配置时的安全兜底；
- 模板默认影响所有绑定该模板、且没有单独覆盖的角色；
- 当前角色覆盖只影响当前角色；
- 全局周期默认只负责轮数、消息数、读取上限和重叠消息等周期参数。

业务模块不再各自推断优先级。

## 三、表结构工作台

表结构与记忆策略页面新增两个设置范围：

```text
模板默认
当前角色覆盖
```

模板默认模式：

- 可以修改表格结构、字段结构与模板策略；
- 保存后影响所有绑定该模板、且没有角色覆盖的角色。

当前角色覆盖模式：

- 只允许修改采集、写入、周期、更新权限和召回策略；
- 模板名称、表格名称、字段结构和字段级策略保持只读；
- 未覆盖的项目继续继承模板或全局默认；
- 每张表可以单独“恢复模板”。

## 四、当前生效值与来源

每张表新增：

- 当前生效；
- 来源。

当前生效会直接显示系统实际使用的：

- 信息采集方式；
- 写入方式；
- 最终周期；
- 召回方式。

来源会分别标记：

```text
当前角色
模板
全局
系统
```

因此能够直接回答：

- 为什么这张表会自动整理；
- 为什么它需要审核；
- 当前周期来自模板还是全局；
- 为什么它不会被召回。

## 五、业务执行统一

有效策略解释器已接入：

- 周期整理到期判断；
- 更新范围计算；
- AI 更新 Prompt；
- AI 直接写入与审核分流；
- 字段级写入策略继承；
- Sidecar 目标表选择；
- 短期候选采集；
- 记忆召回与 Prompt 注入；
- 表格运行状态展示。

旧的隐藏自动化选择器如果仍被兼容界面调用，也只会写入当前角色覆盖，不再修改模板本身。

## 六、字段策略与表策略闭合

R3 的字段级写入策略允许字段选择“继承表格”。本版修复了一个容易忽略的断点：

- 审核草案；
- AI 直接更新；
- 已接受审核的写入；

现在都会先取得当前角色的有效表格策略，再判断字段最终是直接写入、进入审核、进入候选、仅运行态或仅人工。

因此当前角色把某张表从“先确认”覆盖为“直接生效”时，继承表格的字段会使用同一个最终结果，不再出现界面显示和字段路由不一致。

## 七、数据边界

当前角色覆盖保存在：

```text
chat.memoryTables.policyOverrides[templateId][tableId]
```

它不会修改：

- 模板结构；
- 字段 ID；
- 表格 ID；
- 正式记忆内容；
- 正式记忆历史。

删除某张表的角色覆盖后，会立即重新继承模板默认。

## 九、单人版边界

本版不引入配置审批、多人权限、远程策略发布或环境级配置中心。

当前角色覆盖只是本地单人使用中的轻量差异层：用于同一个模板绑定多个角色时，让个别角色拥有不同的采集、审核、周期或召回方式。
