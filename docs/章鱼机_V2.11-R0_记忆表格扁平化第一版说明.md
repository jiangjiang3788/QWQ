# 章鱼机 V2.11-R0：记忆表格扁平化第一版

## 1. 本版目标

本版以 V2.10-R6 为基线，只处理档案记忆的界面与渲染收敛：

- 进入记忆页后尽快看到真实表格；
- 删除重复的旧卡片渲染路径；
- 普通模式改为阅读优先，只有选中的行或字段才创建编辑器；
- 每行大量常驻按钮收敛为一个操作菜单；
- 自动整理、游标和表级调度移入右侧抽屉；
- 桌面端和手机端共用同一套表格渲染与状态；
- 不新增一层首页或卡片外壳。

本版不修改记忆 schema、模板格式、备份格式、模型 Prompt、召回、审核或任务队列逻辑。

## 2. 运行时实际效果

使用用户提供的清理版 `.ee` 备份，在受控浏览器运行壳中加载真实的八张记忆表与实际数据。

本次截图选中“待办、承诺与未完成事项”，实际渲染 15 行。

## 3. 用户可见改动

### 3.1 表格成为页面主体

记忆页保留一条轻量角色状态、工作区导航和搜索工具栏，随后直接进入：

```text
左侧表清单 / 手机端横向表标签
                    ↓
真实键值表或行表
```

不再先显示整页自动整理配置。

### 3.2 普通模式改为只读

- 行表单元格默认显示格式化文本；
- 键值档案默认显示格式化值；
- 点击“编辑”后，只为当前行或当前字段创建编辑器；
- 新增行后自动进入该行编辑状态；
- 完成编辑后立即恢复阅读模式。

### 3.3 行操作收敛

原来每行常驻的编辑、移动、删除、标签、来源、暂停、固定等按钮，统一收进一个“操作”选择菜单。

标签、固定和暂停状态仍以轻量标记展示，不删除原能力。

### 3.4 整理设置抽屉化

“整理设置”按钮打开右侧抽屉，包含：

- 全局自动整理；
- 各表整理通道；
- 目标表和消息位置；
- 保存、整理、预览、补救与跳到最新等操作。

关闭抽屉后，表格恢复全部宽度。手机端使用不超过 94vw 的同一抽屉。

## 4. 代码收敛

### 新增复用模块

```text
js/features/memory/table_view.js
```

统一负责：

- 文本和值的安全展示；
- 空值、状态、标签和元信息展示；
- 行操作菜单渲染；
- HTML 转义。

控制器不再自行拼接多套展示结构。

### 新增统一样式

```text
css/modules/memory_table_flat.css
```

集中维护：

- 扁平工作区；
- 桌面左侧表清单；
- 手机横向表标签；
- 表格阅读与编辑状态；
- 整理设置抽屉；
- 日夜模式适配。

### 删除旧生产路径

从 `memory_table.js` 移除：

- `renderKeyValueFieldCard()`；
- `renderRowsTableCard()`；
- `renderFieldChartContainer()`；
- `getFieldHistorySeries()`；
- `drawAllCharts()`；
- `drawSparkline()`；
- 已无调用的字段筛选与条件着色导入。

`memory_table.js` 从 4,748 行降至 4,547 行。本版先删除确认无调用的重复路径；后续再按 controller、grid、update service、context assembler 分批拆分，避免一次性大重写。

## 5. 数据与兼容边界

本版没有修改：

- IndexedDB schema；
- `memoryTables.data`；
- 模板和字段定义；
- `row.meta` 与 `tagBundle`；
- 结构化记忆审核批次；
- 任务队列和游标；
- 召回、注入与 Sidecar；
- `.ee` 完整/精简备份；
- Provider 请求和模型调用次数。

旧数据不需要迁移，可直接使用当前备份恢复。

## 7. 第一版明确未做

以下内容属于后续版本，不在本版混入：

- 大模型自动生成和归并 `tagBundle`；
- 整理目标表时召回其他相关表；
- 跨表重复、冲突和合并动作；
- 记忆详情侧栏；
- 正式数据 schema 迁移；
- 将 `memory_table.js` 一次性拆成大量小文件。

下一版应在当前扁平 UI 基础上先拆出 `update_service` 与 `context_assembler`，再接入模型标签和相关表上下文，避免把新逻辑重新堆回主控制器。
