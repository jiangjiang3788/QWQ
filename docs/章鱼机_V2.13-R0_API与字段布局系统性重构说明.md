# 章鱼机 V2.13-R0：API 与字段布局系统性重构说明

## 一、问题根因

### 1. KV 字段名称列过窄

原实现虽然已经有 `field_width.js`，但 KV 场景仍使用较小的估算参数，桌面上限只有 116px、手机上限只有 76px。混合英文、下划线和中文的字段名会被低估，并被 `overflow-wrap:anywhere` 强制拆行，因此同一列视觉上显得窄且凌乱。

### 2. API 管理存在两套导航

页面已有“聊天 / 记忆 / 自动化 / 感知”工作区导航，`api_settings_ui.js` 又动态生成“主聊天 / 总结 / 向量 / 后台 / 人设 / 识图”按钮，形成重复层级。第二套按钮只是在同一页面内跳转，没有独立业务价值。

### 3. 向量 API 配置后仍不能工作

原流程存在四个结构性问题：

1. 向量配置缺失时会回退到总结或主聊天 API，但聊天模型通常不支持 Embeddings。
2. 服务商和模型选择允许直接选中聊天模型，保存时又不发送真实向量请求。
3. 两个向量使用方分别实现请求、批处理和解析，端点及错误处理容易漂移。
4. API 地址统一追加 `/v1/embeddings`，当用户已填写到 `/v1` 时可能出现 `/v1/v1/embeddings`。

## 二、重构方案

### 1. 新增统一 API 服务注册表

新增 `js/core/api_service_registry.js`，统一负责：

- API 角色与数据库配置映射；
- 各角色允许的回退规则；
- OpenAI 兼容与 Gemini 协议识别；
- `/models`、`/embeddings`、`/chat/completions` 端点规范化；
- 模型列表读取与向量模型候选排序；
- Embedding 请求、批处理、返回数量和维度校验；
- 向量配置健康状态。

向量角色的回退列表为空。未通过真实 Embedding 测试的旧配置、待验证配置和失败配置均不会参与语义检索。

### 2. 向量 API 改为“测试成功后启用”

保存向量配置时发送一条真实 Embedding 请求。成功后保存：

- `enabled: true`
- `health: ready`
- 验证时间
- 实际向量维度
- 请求耗时

失败时保存错误原因并设为不可用，检索自动退回关键词模式，不再反复调用错误接口。

### 3. 合并向量请求实现

`vector_memory.js` 与 `memory_table_retrieval.js` 均调用 `OVOApiServiceRegistry.embed()`，删除各自的 `/v1/embeddings` 拼接、批处理和返回解析代码。

### 4. 删除重复 API 子导航

退役 `api_settings_ui.js` 动态生成的“主聊天 / 总结 / 向量 / 后台 / 人设 / 识图”按钮及对应 CSS。页面只保留“聊天 / 记忆 / 自动化 / 感知”一套导航。

### 5. KV 字段列按最长字段名统一测量

`field_width.js` 使用 Canvas `measureText()` 按实际界面字体测量当前表所有可见字段名，以最长字段名决定整列宽度：

- 桌面安全范围：116–260px；
- 手机安全范围：92–172px；
- 同一表的所有字段名单元格使用同一个 CSS 变量；
- 超过安全上限时单行省略，悬停或查看 title 可读取完整名称。

## 三、兼容与迁移行为

- 原聊天、总结、后台、人设与识图配置继续读取。
- 原向量配置会显示“待验证”，必须点击“测试并保存向量 API”后才启用。
- OpenAI 兼容地址可填写到域名根路径、`/v1`、`/v1/models` 或 `/v1/embeddings`，注册表会规范化，不重复添加 `/v1`。
- 旧压缩包中的 `#Uxxxx` 文件名已统一还原为正常中文 Unicode 文件名。

## 四、验证结果

- 全部 JavaScript 文件通过 `node --check`。
- `python3 tests/run_static_checks.py`：PASS。
- `python3 tools/check_memory_architecture.py`：PASS。
- 54 个 Node 回归测试全部通过。
- 新增 API/向量架构静态测试和统一向量传输运行时测试。

## 五、实际使用步骤

进入“API 管理 → 记忆 → 向量记忆 API”，填写支持 Embeddings 的网关地址、密钥和向量模型。先拉取模型，再选择真正的 Embedding 模型，最后点击“测试并保存向量 API”。只有状态显示“已验证 · N 维”后，混合语义检索才会启用。
