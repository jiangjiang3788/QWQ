# 章鱼机 V2.13-R5.3：记忆策略与表格设置收敛说明

## 目标

本版将模板结构、逐表更新通道、逐表写入审核、召回策略和当前角色运行状态收敛到同一个全屏表格工作台。日常界面不显示表 ID 或字段 ID；内部 ID 仅保留在折叠的原始 JSON 中。

本版只完成设置与策略收敛，不包含 V2.13-R5.4 计划中的“更新审核批次原子事务重写”。长期候选晋升继续使用 V2.13-R5 已完成的原子链路。

## 一、统一后的设置边界

### 1. 记忆运行设置

侧栏只保留真正跨表的全局配置：

- 聊天记忆采集：是否跟随正常 AI 回复采集、是否保存短期候选、是否显示聊天状态提示；
- 周期整理默认值：总开关、默认触发方式、默认轮数、默认消息数、默认读取上限；
- 召回与安全：检索方式、语义与标签权重、向量候选数、场景路由和副作用保护。

已移除重复入口：

- 全局审核方式；
- 抽屉中的逐表更新通道；
- 抽屉中的逐表游标和“整理此表”。

### 2. 表结构与记忆策略工作台

每张表在一行中配置：

- 表格名称、职责、形态、层级；
- 信息来源、写入方式、调用 API、频率来源；
- 触发条件、轮数、消息数、读取上限和重叠消息；
- 新增、修改、删除权限；
- 注入模式、Top-K、阈值、预算、有效期；
- 提取、更新和注入规则；
- 未处理消息、上次运行、待确认状态和游标。

表格使用连续细分隔线布局。输入控件默认透明无边框，聚焦时才显示编辑边界。

## 二、新的稳定策略字段

### systemRole：表格职责

- `general`：普通记忆表
- `core_profile`：核心档案
- `current_state`：当前状态
- `tasks`：待办事项
- `recent_events`：近期经历
- `daily_observation`：日常观察
- `medium_summary`：中期总结
- `long_candidate`：长期候选
- `long_store`：稳定长期库

聊天即时更新、短期候选保存和长期晋升优先依据职责路由，不再只依赖可编辑的中文表名或数组顺序。

### capturePolicy：信息来源

- `sidecar`：聊天同请求，不额外调用 API；
- `scheduled`：按周期整理；
- `manual`：只在用户明确操作时整理；
- `disabled`：关闭自动或模型采集。

周期表另行指定：

- `frequencySource: global`：使用全局默认；
- `frequencySource: table`：使用本表周期；
- `apiMode: main / summary / none`：调用方式。

### commitPolicy：写入方式

- `direct`：直接进入正式表；
- `review`：先生成更新草案，确认后进入正式表；
- `candidate`：进入短期候选池；
- `manual_only`：仅人工编辑或其他受控事务可写；
- `promotion`：批准后晋升到显式长期目标。

是否审核由目标表的写入方式决定，不再由使用主聊天 API 还是总结 API 决定。旧存档缺少新字段时，仍可读取历史全局审核设置作为兼容回退。

## 三、职责冲突门禁

以下职责在同一模板或同一角色绑定范围内必须唯一：

- 核心档案；
- 当前状态；
- 待办事项；
- 近期经历；
- 日常观察；
- 中期总结；
- 长期候选；
- 稳定长期库。

发生重复时，工作台将对应行标红并禁止保存，避免 Sidecar 或晋升逻辑继续使用“第一张匹配表”。普通记忆表允许多张。

## 四、旧模板兼容

缺少新字段的旧模板会按以下信息推导：

1. 已存在的显式新策略；
2. 旧 `automationMode`；
3. `updatePolicy`；
4. 表 ID、表名和记忆层级。

打开工作台不会立即改写模板。只有用户点击保存时，才将显式 `systemRole`、`capturePolicy` 和 `commitPolicy` 写回。原有模板、表、字段和行 ID 不变，正式记忆数据不迁移、不清空。

## 五、默认八张表的策略

| 表格 | 职责 | 信息来源 | 写入方式 |
|---|---|---|---|
| 核心确认档案 | 核心档案 | 手动整理 | 仅人工编辑 |
| 当前状态（3–7天） | 当前状态 | 聊天同请求 | 直接生效 |
| 待办、承诺与未完成事项 | 待办事项 | 聊天同请求 | 直接生效 |
| 近期经历、想法与重要事件 | 近期经历 | 聊天同请求 | 进入候选 |
| 日常观察 | 日常观察 | 聊天同请求 | 进入候选 |
| 中期总结与成长经验 | 中期总结 | 手动整理 | 先确认再生效 |
| 长期候选审核队列 | 长期候选 | 手动整理 | 批准后晋升 |
| 稳定长期特征库 | 稳定长期库 | 关闭模型采集 | 仅人工或晋升事务写入 |

## 六、仍留给 V2.13-R5.4 的事项

- 一个更新审核批次严格限定一张目标表；
- 更新审核应用、拒绝、游标推进和持久化使用同一原子事务；
- 保存失败恢复正式表、审核批次、游标和任务状态；
- 审核 UI 按“记忆记录”分组，而不是按字段建议计数；
- 明确区分“拒绝并跳过消息范围”与“取消并保留消息范围”。
